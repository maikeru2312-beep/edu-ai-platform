# リリース計画（RELEASE PLAN）

最終更新: 2026-06-28（初稿）

> **重要：本計画は「案」。実際の公開・販売は、各項目をユーザーが確認・承認してから行う。** 本リポジトリ上の作業は原稿作成までで止める。

---

## 0. 公開前の絶対条件（ゲート）

以下がすべて満たされるまで、いかなる商品も公開・販売しない。

- [ ] **服務・兼職兼業の確認**（公立教員の場合）。所属自治体の規程・許可要否をユーザー本人が確認。→ [COMPLIANCE_CHECK.md](COMPLIANCE_CHECK.md)
- [ ] 全原稿の人間レビュー（実在児童・学校・自治体が推測される記述ゼロ）
- [ ] 個人情報を入力させるプロンプト・テンプレがないことの確認
- [ ] 収益保証・断定表現がないことの確認
- [ ] 価格の最終決定
- [ ] 販売プラットフォーム（note / BOOTH等）のアカウント・規約確認
- [ ] AdSense審査への影響がないこと（本番サイトのコード・設定を一切触っていない）

---

## 1. 公開順序（推奨）

実用品の完成度と集客導線を踏まえ、**入口→低価格→実用→本命**の順で段階公開する。

| 順 | 商品 | タイミング案 | 公開の狙い | 前提 |
|---|---|---|---|---|
| 1 | A 無料PDF | 最初 | リスト獲得・信頼の入口を先に置く | 原稿確定・配布手段（note無料記事 or 自サイト配布）確定 |
| 2 | B 有料note | Aの数日〜2週間後 | 小さな有料体験を提供し反応を見る | A内でBを予告済み |
| 3 | C 有料PDF | Bの反応を見て | 実用品として本格販売 | BOOTH等の販売設定確定 |
| 4 | D スターターキット | C購入者の声を踏まえ | 本命の上位商品 | C実績・FBを反映してMVPを拡張 |

> 4商品すべての**初稿（フォルダ＋ドラフト）は今回作成する**が、公開は1から順に、各ゲート通過後に行う。

## 2. 配布・販売チャネル案

| 商品 | 配布/販売 | 補足 |
|---|---|---|
| A 無料PDF | note無料記事内リンク / 自サイトの固定ページ配布 | メール登録と引き換えにする場合は個人情報保護方針を明記 |
| B 有料note | note有料記事 | note内課金で完結。手数料・規約を事前確認 |
| C 有料PDF | BOOTH（ダウンロード販売） | 価格・特商法表記・返金方針を確認 |
| D スターターキット | BOOTH（ダウンロード販売） | C購入者向け割引導線を検討（任意） |

※ どのチャネルも**アカウント設定・規約同意・特定商取引法に基づく表記**はユーザーが行う。自動投稿はしない。

## 3. 集客導線（既存資産との接続）

- 教育DXナビの既存記事（例：AI校務改善入門、AIサービスチェックリスト、特別支援ICT・合理的配慮）から、**記事末尾の自然なリンク**で無料PDFへ誘導する案。→ 具体案は [launch/site-linking-plan.md](launch/site-linking-plan.md)
- X：商品紹介ではなく「気づき・チェック観点」を投稿し、興味を持った層を無料PDFへ。→ [launch/x-posts-product-launch.md](launch/x-posts-product-launch.md)
- note：無料PDF配布記事 → 有料note → BOOTHの段階導線。

> 既存記事の**本文・コード・メタ情報（canonical / sitemap / robots / GA4 / AdSense）は変更しない**。リンク追加を行う場合も、別途ユーザー承認のうえ最小限にとどめる。

## 4. 公開後にやること（案・参考）

- 反応（閲覧数・購入数・問い合わせ）を見て価格・タイトルを調整
- 無料PDFのフィードバックを有料商品に反映
- 商品Dは「校内研修で使った」等の声を集めてから拡張

---

## 5. やらないこと（このフェーズ）

- 本番サイトへの販売ページ公開
- note / BOOTH への自動投稿
- AdSense / GA4 / Search Console / robots / sitemap / canonical の変更
- 価格や公開可否のユーザー確認を飛ばした公開
