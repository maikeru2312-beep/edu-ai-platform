# 校務プロンプト50（スターターキット同梱）

> **共通ルール**：個人情報（実名・成績・障害名・診断・家庭環境）は入力しない。固有名は「○○」、子どもは「ある児童」に。出力は必ず教員が確認・修正してから使う。実態の判断・評価・配慮の決定はAIに任せない。校内・自治体の方針が優先。

本キットの50個は「**汎用30 ＋ 特別支援・校内運用20**」で構成。汎用30は単体商品『生成AI校務プロンプト30選』と同じ体系（→ [../note-ai-prompts-30/prompts-30.md](../note-ai-prompts-30/prompts-30.md)）。本ファイルでは**カテゴリ早見＋増補20個（31〜50）**を全文掲載する。

---

## 汎用30（早見・本体は30選を同梱）

| # | カテゴリ | 例 |
|---|---|---|
| 1–6 | 通知・連絡文 | 行事お知らせ／催促のやわらか表現／整文／お礼／欠席案内／やさしい日本語化 |
| 7–10 | 学級通信・学年だより | 箇条書き→本文／季節あいさつ／お願い文／構成案 |
| 11–15 | 授業案・指導案 | 指導案たたき台／導入アイデア／発問例／板書計画／振り返り項目 |
| 16–19 | 教材案 | 練習問題／かみくだき説明／難易度調整／三択クイズ |
| 20–23 | 所見・記録の言い換え | メモの整文／表現バリエーション／成長視点に言い換え／面談メモ整理 |
| 24–27 | 研修・会議資料 | 研修構成案／会議メモ整理／提案骨子／アンケート設問 |
| 28–29 | 保護者向け文面 | 配慮が要る連絡／問い合わせ返信整え |
| 30 | 特別支援の注意 | 一般的な支援選択肢出し（判断はしない） |

## 増補20（31〜50）— 特別支援・校内運用向け

> 各プロンプトに ［使う場面］／［入力NG］／［確認］ を付す。

### I. 特別支援の教材・場面づくり（31〜37）

**31. 視覚的な手順カードの文案**
```
「　（活動）　」の手順を、短い文・1文1動作で、視覚支援カード用に5ステップに分けてください。
```
［使う場面］手順の見える化。［入力NG］特定児童の情報。［確認］言葉の平易さ・順序。

**32. 絵カード/シンボルに添える言葉**
```
「　（場面）　」で使う、短い指示・選択肢の言葉を、やさしい日本語で5つ作ってください。
```
［使う場面］AAC・選択肢提示の言葉づくり。［入力NG］個人情報。［確認］本人に伝わる語彙か。

**33. 教材のやさしい版づくり**
```
次の説明を、（標準／やさしい／とてもやさしい）の3段階に作り分けてください。
説明：「　」
```
［使う場面］個に応じた段階化。［入力NG］なし。［確認］意味の保持・情報の欠落。

**34. 見通しを持たせる予定の言葉**
```
「　（一日/活動の流れ）　」を、見通しが持てるよう短い言葉のスケジュールにしてください。
```
［使う場面］予定提示・構造化。［入力NG］個人情報。［確認］実際の流れと一致。

**35. ほめ言葉・声かけのバリエーション**
```
「　（場面）　」で使える、具体的で前向きな声かけを5パターン作ってください。
```
［使う場面］肯定的な関わりの引き出し。［入力NG］個人情報。［確認］その子に合うか（人が選ぶ）。

**36. クールダウンの選択肢提示文**
```
気持ちが高ぶったときの落ち着く方法の「選択肢」を、本人が選べる短い言葉で5つ挙げてください。
```
［使う場面］選択肢の提示づくり。［入力NG］個人の行動記録。［確認］本人・校内方針に合うか、押し付けでないか。

**37. 一般的な支援アイデアの比較**
```
「　（一般化した困りごと）　」への支援アイデアを、ローテク/ハイテク両方で、利点と留意点つきに挙げてください。特定の子の判断はしないでください。
```
［使う場面］引き出しを増やす。［入力NG］実在児童の情報。［確認］あくまで一般論／決定は人が行う。

### J. 記録・計画の下書き整文（38〜42）

**38. 自分の記録メモの整文**
```
私が書いた支援記録の下書きを、客観的で簡潔な文体に整えてください。事実は変えないこと。
下書き：「　（伏字のメモ）　」
```
［使う場面］記録の文章化。［入力NG］実名・診断。［確認］事実の保持・断定回避。

**39. 目標文の表現案**
```
「　（育てたい力）　」を、個別の指導計画の短期目標らしい表現に3パターンしてください。
```
［使う場面］目標文の言い回し。［入力NG］個人情報。［確認］実態に合うか（人が判断）。

**40. 評価観点の言語化**
```
「　（目標）　」に対する評価の観点と、見取りの例を整理してください。
```
［使う場面］評価設計の下書き。［入力NG］個人情報。［確認］実現可能性・自校様式との整合。

**41. ケース会議の論点整理**
```
次のメモを、ケース会議用に「現状・課題・支援案・確認事項」に整理してください。
メモ：「　（一般化した内容）　」
```
［使う場面］会議準備。［入力NG］実名・診断・家庭の機微情報。［確認］機微情報の扱い・校内手続き。

**42. 引き継ぎ文の下書き**
```
次の引き継ぎメモを、簡潔で誤解の少ない文章に整えてください。事実は変えないこと。
メモ：「　（伏字）　」
```
［使う場面］引き継ぎ資料。［入力NG］実名・診断。［確認］事実保持・受け手にわかるか。

### K. 校内研修・運用（43〜47）

**43. 研修の進行台本**
```
テーマ「　」の45分研修の進行台本（導入・説明・演習・共有・まとめ）を、時間配分つきで作ってください。
```
［使う場面］研修運営。［入力NG］なし。［確認］時間・演習の負担。

**44. 研修の事例（架空）作成**
```
特別支援×ICTの研修で使う、架空の検討事例を1つ作ってください。実在を想起させない設定で。
```
［使う場面］事例検討の素材。［入力NG］実在情報。［確認］実在を想起させないか。

**45. AI利用ルールのたたき台**
```
学校の「生成AI利用の最低限のルール」案を、個人情報・確認・記録の観点で箇条書きにしてください。
```
［使う場面］ルール整備の出発点。［入力NG］なし。［確認］自治体方針との整合（人が確認）。

**46. 研修アンケートの設問**
```
AI・ICT研修後のふりかえりアンケートを、選択式中心で6問作ってください。
```
［使う場面］研修評価。［入力NG］回答者特定設問。［確認］匿名性・誘導回避。

**47. FAQ（よくある不安）への回答案**
```
「学校で生成AIを使うことへのよくある不安」5つと、丁寧な回答案を作ってください。断定はしない。
```
［使う場面］説明資料。［入力NG］なし。［確認］断定回避・校内方針優先の明記。

### L. 保護者・連携（48〜50）

**48. 保護者向け説明文（ICT活用）**
```
ICTを学習に取り入れる旨を保護者に説明する文を、相談・合意の姿勢で作ってください。一般的な内容で。
```
［使う場面］説明文の下書き。［入力NG］個別の機微情報。［確認］通知でなく相談の姿勢／校内手続き。

**49. 関係機関への連絡文の整え**
```
関係機関への連絡文の下書きを、丁寧で簡潔に整えてください。事実は変えないこと。
下書き：「　（伏字）　」
```
［使う場面］連携文書の整文。［入力NG］実名・診断・機微情報。［確認］機微情報の扱い・送付手続き。

**50. 一年の振り返り（自分用メモの整理）**
```
私の実践メモを、来年度の改善につながる形（うまくいった点・課題・来年度試すこと）に整理してください。
メモ：「　（伏字）　」
```
［使う場面］自己省察の整理。［入力NG］実名・機微情報。［確認］個人が特定されないか。
