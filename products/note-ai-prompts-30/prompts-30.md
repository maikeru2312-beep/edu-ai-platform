# 先生のための生成AI校務プロンプト30選（本体）

> **共通ルール（全プロンプト共通）**
> - 個人情報（実名・成績・障害名・家庭環境・住所・連絡先）は入力しない。固有名は「○○」、子どもは「ある児童／生徒」に置き換える。
> - 出力は**必ず教員が読み直し、事実・表現・配慮を確認してから**使う。
> - 児童生徒の実態の判断・評価・見立てはAIに任せない。AIは下書きまで。
> - 校内・自治体の文書ルール／AI利用方針が常に優先。
>
> 各プロンプトは **［使う場面］／［入力してはいけない情報］／［出力後に確認すること］** を添えています。`「　」`部分を自分の状況に書き換えて使ってください。

---

## A. 通知・連絡文（1〜6）

### 1. 行事の保護者向けお知らせ
```
小学校の教員です。保護者向けに「○○（行事名）」のお知らせ文の下書きを作ってください。
条件：日時「　」、場所「　」、持ち物「　」、丁寧で簡潔、A4半分程度、箇条書きを活用。
```
［使う場面］運動会・遠足・参観日などの連絡文の下書き。
［入力NG］保護者・児童の氏名、連絡先、個別事情。
［確認］日時・場所・持ち物の事実、学校の正式名称・締切、配布ルール適合。

### 2. 提出物の催促（やわらかい表現）
```
保護者向けに、提出物「　」の提出をお願いする一文を、責めない・やわらかい表現で3パターン作ってください。
```
［使う場面］未提出のお願いを角が立たないよう伝えたいとき。
［入力NG］未提出者の氏名・人数などの特定情報。
［確認］対象を限定しすぎていないか、催促の口調が強すぎないか。

### 3. 緊急時でない一般連絡の整文
```
次のメモを、保護者に伝わる連絡文に整えてください。事実は変えず、簡潔に。
メモ：「　（箇条書きで状況）　」
```
［使う場面］箇条書きメモを連絡文に整えるとき。
［入力NG］個人が特定される事実、未確定の情報。
［確認］事実の改変がないか、誇張・憶測が混じっていないか。

### 4. お礼・感謝の文面
```
学校行事「　」にご協力いただいた保護者・地域の方へのお礼文を、丁寧で温かい文体で作ってください。200字程度。
```
［使う場面］協力者へのお礼文。
［入力NG］個人名・団体名（必要なら出力後に自分で追記）。
［確認］関係者の呼称・敬称、事実関係。

### 5. 欠席・連絡フォームの案内文
```
保護者向けに、欠席連絡の方法「　（手段）　」を案内する文を、手順がわかるように作ってください。
```
［使う場面］連絡手段の周知。
［入力NG］保護者の個人情報、ログイン情報。
［確認］手順の正確さ、リンク・締切。

### 6. 連絡文の「やさしい日本語」化
```
次の連絡文を、やさしい日本語（短い文・むずかしい言葉を言いかえ・ふりがな配慮）に書き直してください。
文：「　」
```
［使う場面］多様な家庭背景に配慮した連絡。
［入力NG］個人情報。
［確認］意味が変わっていないか、必要な情報が落ちていないか。

## B. 学級通信・学年だより（7〜10）

### 7. 箇条書きから通信本文へ
```
学級通信用に、次の箇条書きを保護者向けの文章にしてください。子どもの様子は一般的な表現で。
箇条書き：「　」
```
［使う場面］メモを通信の文章にするとき。
［入力NG］特定の児童名・個別の評価。
［確認］実際の様子と合っているか、特定の子に偏っていないか。

### 8. 季節のあいさつ文
```
○月の学級通信の冒頭にふさわしい、季節のあいさつ文を3パターン作ってください。各2〜3文。
```
［使う場面］通信の書き出し。
［入力NG］なし（一般文）。
［確認］季節・地域感の違和感、使い回しの不自然さ。

### 9. 家庭へのお願い文
```
家庭に「　（お願い内容）　」を協力してほしい旨を、押しつけがましくない表現でお願いする文を作ってください。
```
［使う場面］家庭学習・持ち物・生活習慣などの協力依頼。
［入力NG］特定家庭の事情。
［確認］全家庭に当てはまる書き方か、強制に聞こえないか。

### 10. 通信の構成案出し
```
○月の学級通信の構成案（見出し3〜4本）を、行事「　」をふまえて提案してください。
```
［使う場面］何を書くか迷うとき。
［入力NG］個別事案。
［確認］学校の公式発信として適切な話題か。

## C. 授業案・指導案（11〜15）

### 11. 単元の指導案たたき台
```
小学校「　（教科・学年）　」の単元「　」について、目標・主な学習活動・評価の観点を含む指導案のたたき台を作ってください。
```
［使う場面］指導案の骨組みづくり。
［入力NG］児童の個別情報。
［確認］学習指導要領・自校の指導計画との整合、活動の実現可能性。

### 12. 授業の導入アイデア
```
「　（教科・題材）　」の授業の導入（つかみ）アイデアを5つ、所要時間の目安つきで提案してください。
```
［使う場面］導入の発想支援。
［入力NG］なし。
［確認］安全面・準備の負担、児童の実態への適合。

### 13. 発問例の作成
```
「　（題材）　」について、子どもの考えを引き出す発問を、易→難の順で5つ作ってください。
```
［使う場面］発問の幅を広げたいとき。
［入力NG］なし。
［確認］発達段階に合うか、誘導的すぎないか。

### 14. 板書計画の案
```
「　（授業内容）　」の板書計画の案を、構造（中心・対比・まとめ）がわかる形で提案してください。
```
［使う場面］板書の整理。
［入力NG］なし。
［確認］情報量、視覚的なわかりやすさ。

### 15. 振り返り（ワークシート）項目案
```
「　（授業）　」の振り返りシートの設問を、自己評価＋記述の形で4問作ってください。
```
［使う場面］振り返り設計。
［入力NG］なし。
［確認］負担が重すぎないか、答えやすい言葉か。

## D. 教材案（16〜19）

### 16. 練習問題のたたき台
```
「　（教科・単元）　」の練習問題を、易しい順に5問作ってください。解答も付けてください。
```
［使う場面］補充・確認問題づくり。
［入力NG］なし。
［確認］解答の正誤、難易度、表記。

### 17. 説明の言い換え（かみくだき）
```
次の説明を、小学生にもわかるやさしい言葉に言い換えてください。たとえ話を1つ添えて。
説明：「　」
```
［使う場面］むずかしい概念の説明。
［入力NG］なし。
［確認］正確さを損なっていないか、たとえが適切か。

### 18. 教材の難易度調整
```
次の文章を、（やさしい版／標準版）の2段階に作り分けてください。
文章：「　」
```
［使う場面］個に応じた教材の用意。
［入力NG］特定児童の情報。
［確認］情報の欠落、語彙レベル。

### 19. クイズ・確認テストの選択肢作成
```
「　（題材）　」の三択クイズを5問、正解と簡単な解説つきで作ってください。
```
［使う場面］確認・導入のクイズ。
［入力NG］なし。
［確認］正解・誤答選択肢の妥当性。

## E. 所見・記録の言い換え（20〜23）

> ⚠️ 所見は**実際に子どもを見た教員が書くもの**。AIは「自分が書いた下書きの表現を整える」用途に限る。子どもの様子そのものをAIに作らせない。

### 20. 自分の所見メモの整文
```
私が書いた所見の下書きを、保護者に伝わる前向きな文体に整えてください。事実は変えないこと。
下書き：「　（固有名を伏せた自分のメモ）　」
```
［使う場面］自分のメモの文章化。
［入力NG］実名、他児との比較、断定的な評価。
［確認］事実と合っているか、紋切り型になっていないか。

### 21. 表現のバリエーション出し
```
「　（伝えたい良さ）　」を表す所見の言い回しを、決まり文句にならないよう5パターン作ってください。
```
［使う場面］似た表現の繰り返しを避けたいとき。
［入力NG］実名・個別事情。
［確認］その子に本当に当てはまるか。

### 22. ネガティブを成長視点に言い換え
```
次の課題を、否定的でなく「これからの成長」として書く表現に言い換えてください。
課題：「　（一般化した内容）　」
```
［使う場面］課題を前向きに伝えたいとき。
［入力NG］実名、家庭の問題。
［確認］事実を美化・歪曲していないか。

### 23. 面談メモの整理
```
保護者面談で話す要点を、次のメモから整理してください（伝える順番つき）。
メモ：「　（一般化した内容）　」
```
［使う場面］面談準備。
［入力NG］実名、要配慮個人情報。
［確認］機微な内容の扱い、伝え方の配慮。

次の会議メモを、決定事項・宿題・次回までにやることに分けて整理してください。

### 27. アンケート設問づくり

## G. 保護者向け文面（28〜29）

### 28. 配慮が必要な連絡の下書き
```
保護者に「　（一般化した連絡）　」を、丁寧で誤解の生じにくい表現で伝える文を作ってください。
```
［使う場面］伝え方に気をつかう連絡。
［入力NG］個別の機微情報、診断・評価。
［確認］断定を避ける、学校の方針と一致。

### 29. 問い合わせへの返信文の整え
```
保護者からの問い合わせ「　（要旨）　」への返信文を、丁寧で簡潔な文体で整えてください。約款的な断定はしない。
```
［使う場面］返信文の下書き。
［入力NG］実名、やり取りの個人情報。
［確認］約束しすぎ・断定をしていないか、管理職確認の要否。

## H. 特別支援教育で使うときの注意（30）

### 30. 支援アイデアの一般的な選択肢出し（※判断はしない）
```
「　（一般的な学習場面・困りごとを抽象化）　」に対して、ICTや教材で考えられる支援の一般的な選択肢を、メリットと留意点つきで挙げてください。特定の子の診断や評価はしないでください。
```
［使う場面］支援の引き出しを増やす発想支援。
［入力NG］**実在児童の氏名・障害名・診断・家庭環境・行動記録**。要配慮個人情報は一切入力しない。
［確認］
- 出力はあくまで**一般的な選択肢**。実際の支援は本人・保護者・関係機関と相談し、所属校・教育委員会の方針に従って決める。
- **子どもの理解・見立て・合理的配慮の決定をAIに代行させない。**
- 個別の指導計画・記録への記載は、教員が責任を持って自分の言葉で行う。

> 特別支援向けの印刷して使えるチェックリスト・記入式シートは、有料PDF『特別支援教育×ICT・AI活用チェックリスト集』に収録予定です。
