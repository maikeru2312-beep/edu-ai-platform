「Microsoft Copilotって何ができるの?」「ChatGPTとどう違うの?」「学校のパソコンで使えるの?」——そうした疑問を持つ教員の方は多いのではないでしょうか。

Microsoft Copilot（マイクロソフト コパイロット）は、Microsoftが提供する生成AIサービスです。Word・Excel・PowerPoint・Teamsなど、多くの学校で日常的に使われているMicrosoft 365アプリと連携できる点が特徴です。

ただし、**学校現場で使う際には「便利さ」だけでなく「安全性」と「ルールの確認」が欠かせません**。本記事では、Copilotの基本から校務での活用例、ChatGPTとの違い、セキュリティ上の注意点までを整理します。

> Microsoft Copilotの利用については、**所属校・教育委員会・自治体のガイドラインを必ず事前に確認**してください。所属校・自治体・管理者の設定によって利用範囲が異なります。本記事は一般的な情報の整理であり、特定のサービス利用を推奨するものではなく、個別の法的・行政的判断を代行するものでもありません。

## Microsoft Copilotとは

Microsoft Copilot（以下、Copilot）は、Microsoftが提供する**対話型の生成AIサービス**です。大きく分けて次の2つの利用形態があります。

### Web版Copilot（旧 Bing Chat）

ブラウザから [copilot.microsoft.com](https://copilot.microsoft.com) にアクセスして利用する形態です。Microsoftアカウントがあれば基本的な機能を利用できます。

- テキストでの質問・回答
- 文章生成・要約・翻訳
- 画像生成（一部機能）
- Web検索との連携

Web版Copilotは、ChatGPTと同じように「チャット画面でAIに質問する」使い方です。ブラウザがあればすぐに試せるため、まずCopilotがどのようなものか体験してみたい場合はこちらから始めるとよいでしょう。

### Microsoft 365 Copilot（アプリ連携版）

Word・Excel・PowerPoint・Outlook・Teamsなどの**Microsoft 365アプリ内で動作する**AI支援機能です。文書やデータを直接参照しながらAIが支援してくれるため、校務文書の作成・編集で特に便利です。

たとえば、Wordで文書を開いた状態でCopilotに「この文章を要約して」と依頼すれば、別のサービスにコピー&ペーストすることなく、Word内で作業が完結します。

ただし、**Microsoft 365 Copilotは追加のライセンスが必要**であり、学校・自治体で導入されているかどうかは組織ごとに異なります。「自分の学校で使えるかどうか」は、管理者やICT担当に確認してください。

### 2つの形態の違い

| | Web版Copilot | Microsoft 365 Copilot |
|---|---|---|
| **アクセス方法** | ブラウザでcopilot.microsoft.comへ | Word・Excel等のアプリ内 |
| **追加ライセンス** | 基本機能は不要 | 必要（組織契約） |
| **ファイル参照** | 不可（テキスト入力のみ） | 開いているファイルを参照可能 |
| **主な用途** | 汎用的な質問・文章生成 | アプリ内での文書作成・編集支援 |

### Copilotは常に進化している

Copilotの機能・対応範囲・プラン体系は頻繁にアップデートされています。Microsoftは新機能の追加や名称の変更を定期的に行っており、本記事の内容はあくまで執筆時点の概要です。最新情報はMicrosoft公式サイトで確認してください。

## ChatGPTとの主な違い

「ChatGPTとCopilotは何が違うのか?」は、よく聞かれる質問です。ここでは一般的な比較を整理しますが、**どちらが優れているかは用途や環境によって異なり**、一概に優劣をつけられるものではありません。

| 比較項目 | ChatGPT（OpenAI） | Microsoft Copilot |
|---|---|---|
| **提供元** | OpenAI | Microsoft |
| **基本的な使い方** | Webブラウザ・スマホアプリ | Webブラウザ・Microsoft 365アプリ |
| **AIモデル** | OpenAI独自モデル（GPT系列） | OpenAIモデルを含む複数のモデル |
| **強み** | 汎用的な文章生成・対話 | Microsoft 365アプリとの連携 |
| **Web検索** | 対応（モデル・プランによる） | 対応（Bing連携） |
| **料金** | 無料プランあり・有料プランあり | 無料利用枠あり・有料プランあり |
| **法人向け** | ChatGPT Enterprise / Team | Microsoft 365 Copilot（組織契約） |

### 教員にとっての違いのポイント

- **WordやExcelを日常的に使っている場合**、Copilotはアプリ内から直接AIを呼び出せるため、ファイルのコピー&ペーストの手間が減ります（Microsoft 365 Copilotライセンスが必要）
- **汎用的な文章作成・アイデア出し**には、ChatGPTもCopilotも活用できます
- **どちらを使うかは、所属校・自治体が認めている環境に依存します**。許可された環境の中で選択してください

両サービスの料金体系・機能は頻繁に変更されるため、具体的なプラン名・価格の記載は割愛します。最新情報は各公式サイトを確認してください。

ChatGPTの基本的な使い方について詳しく知りたい場合は、[教員のためのChatGPT入門](/articles/chatgpt-teacher-beginner-guide)もあわせてご覧ください。

## 教員が校務で使いやすい場面

Copilotを校務で活用する場面は多くありますが、**個人情報を含まない業務**から始めることを強くお勧めします。以下は、比較的取り組みやすい例です。

| 業務カテゴリ | 活用例 |
|---|---|
| **文書作成** | 学年だより・学級通信の下書き、保護者向けお知らせ文の作成 |
| **授業準備** | 導入活動のアイデア出し、ワークシートの構成案 |
| **資料作成** | 研修資料・プレゼンのたたき台、配付資料の構成整理 |
| **情報整理** | 会議メモの要点整理、校内文書の要約 |
| **翻訳・言い換え** | 外国語話者の保護者向け文書の翻訳補助、文章の平易化 |
| **事務作業** | 簡単な表計算の関数相談、データ整理のアイデア |

いずれの場合も、**AIの出力をそのまま使うのではなく、教員が内容を確認・修正してから利用する**ことが大前提です。

### まず避けるべき使い方

逆に、以下のような使い方は避けてください。

- **個人情報を含むファイルを開いた状態でCopilotを使う**（成績一覧、出欠記録、個人情報台帳など）
- **児童生徒の氏名や家庭情報をプロンプトに入力する**
- **生成された文章をそのまま公的文書として配布する**（必ず内容確認・修正を行う）
- **AIの出力を成績評価や指導要録の最終判断に使う**

「安全な範囲で試す → 結果を確認する → 少しずつ活用範囲を広げる」というステップが現実的です。

校務でのAI活用全般については、[AIで変わる校務改善](/articles/ai-koomu-kaizen-nyumon)で詳しく紹介しています。

## Wordでの活用例

Word上でCopilotを使うと、文書の下書き・編集・要約をアプリ内で直接行えます。以下は、**個人情報を含まない範囲**での活用例です。

### 学年だよりの下書き

**依頼例：**

```
7月の学年だより（小学4年生向け）のリード文を200字程度で書いてください。
内容：1学期のまとめ、夏休みの過ごし方の呼びかけ、保護者へのお礼
```

生成された文章はあくまでたたき台です。学校の実態や学年の状況に合わせて修正してください。

学級通信の作成にAIを活用するプロンプト例は、[AI学級通信プロンプト](/articles/ai-class-newsletter-prompt)でも紹介しています。

### 文章の要約・書き換え

長い文章を短く要約したり、硬い表現をやわらかく言い換えたりする作業は、Wordの中でCopilotに依頼できます。

**依頼例：**

```
以下の文章を、保護者にとってわかりやすい表現に書き換えてください。
「本校の教育課程において、情報活用能力の育成を体系的に推進してまいります。」
```

### 会議資料のドラフト

校内研修や保護者会の配付資料の構成案を作成できます。

**依頼例：**

```
校内研修「タブレット端末の活用ルール見直し」の配付資料の構成案を作ってください。
対象：全教職員　時間：45分　形式：A4で1枚程度
```

### 文体の統一・校正

複数の教員が分担して書いた文書の文体を統一したり、誤字脱字のチェックを依頼したりすることもできます。

**依頼例：**

```
以下の文章の文体を「です・ます調」に統一し、誤字脱字があれば修正してください。
```

これは特に、学年便りや行事のお知らせなど、複数の先生が原稿を持ち寄る場面で役立ちます。

### 注意点

- **WordにCopilotが表示されない場合**は、Microsoft 365 Copilotライセンスが組織に導入されていない可能性があります。個人判断でライセンスを追加購入するのではなく、管理者・ICT担当に確認してください
- Copilotが参照するのはWordファイルの内容です。**児童生徒の名前・成績・家庭情報が含まれるファイルでCopilotを使用することは避けてください**

## Excelでの活用例

ExcelでのCopilot活用は、データ分析・関数の提案・グラフ作成などに活用できます。ただし、**個人情報を含むデータ（成績一覧・出欠記録・個人情報台帳など）でのCopilot利用は避けてください**。

### 関数・数式の相談

Excel関数に詳しくなくても、やりたいことを日本語で伝えれば、適切な関数を提案してもらえます。

**依頼例：**

```
B列に点数、C列に合否を表示したい。70点以上なら「合格」、それ未満なら「再テスト」と表示する関数を教えてください。
```

### データの整理・集計

行事の参加者数集計、教材費の計算、時間割の整理など、個人を特定しないデータの整理に活用できます。

**依頼例：**

```
この表の月別合計と平均を追加してください。
```

### グラフの作成支援

データをもとにどのようなグラフが適切かを相談し、作成を支援してもらえます。

**依頼例：**

```
この消耗品の月別支出データを、見やすい棒グラフにしてください。
```

### 条件付き書式の設定相談

特定の条件に応じてセルの色を変えるなど、条件付き書式の設定方法を相談することもできます。

**依頼例：**

```
D列の数値が100を超えた場合にセルを赤色にする条件付き書式の設定手順を教えてください。
```

### 注意点

- **成績データ・出欠データ・個人情報を含むExcelファイルでは、Copilotの利用を避けてください**。Copilotに入力されたデータの取り扱いは、組織の設定・契約プランによって異なります
- 関数やマクロの提案は便利ですが、**生成されたコードは必ず動作確認を行ってください**。意図しない計算結果になる場合があります
- 重要なファイルでは、Copilotを使う前にバックアップを取ることをお勧めします
- 数式の結果は必ずサンプルデータで検証し、正しい計算結果が出ることを確かめてから本番データに適用してください

## PowerPointでの活用例

PowerPointでCopilotを使うと、スライドの構成案作成・デザイン提案・内容の要約が可能です。

### 研修資料のスライド構成

**依頼例：**

```
「情報モラル教育の進め方」をテーマに、教員向け校内研修（30分）のスライド構成案を作成してください。8〜10枚程度でお願いします。
```

### 既存資料の要約

長い資料の内容をスライドに要約する際に活用できます。

**依頼例：**

```
この文書の内容を5枚のスライドに要約してください。各スライドにタイトルと箇条書き3〜4点でまとめてください。
```

### 保護者会の説明資料

**依頼例：**

```
保護者会で配付する「家庭でのタブレット利用ルール」の説明スライドを作成してください。
対象：小学校中学年の保護者　枚数：6枚程度
内容：利用時間の目安、フィルタリング、家庭での約束づくり
```

### 注意点

- 生成されたスライドのデザイン・レイアウトは、学校の既存テンプレートに合わないことがあります。体裁の調整は手動で行ってください
- **スライドに含まれる情報の正確性は、必ず教員が確認してください**。特に統計データや制度の説明は、公式資料と照合することが重要です
- 著作権で保護されたイラスト・写真がAIによって使用される場合があるため、画像の出典と利用権には注意してください

## Teamsでの活用例

Microsoft Teamsは多くの学校で連絡・会議ツールとして利用されています。Teams内でもCopilotが活用できる場面があります。

### 会議の要約

Teams会議の録画・文字起こし機能と連携して、会議内容の要約を作成できます。

**活用例：**

- 職員会議後に議論のポイントを箇条書きで整理する
- 研修の内容を要約して、欠席者に共有する
- 次回のアクションアイテムをリストアップする

### チャットの要点整理

長いチャットのやり取りがあった場合に、要点をまとめてもらう機能があります。

**活用例：**

- 学年チームの長いチャット履歴から決定事項を抽出する
- 複数のスレッドにまたがる情報を整理する

### メール（Outlook）での下書き支援

Teamsと同様にOutlookでもCopilotを活用できます。保護者向けの一斉メールや、外部機関への連絡文など、定型的なメールの下書きに利用できます。

**活用例：**

- 保護者向けの行事案内メールの下書きを作成する
- 他校との連携行事に関する連絡文の構成を整える
- 英語でのメール返信の下書きを作成する（外国人保護者への対応など）

ただし、メールの本文に児童生徒の個人情報を含めた状態でCopilotを使用することは避けてください。

### 注意点

- Teams内のCopilot機能は、**組織のMicrosoft 365設定・ライセンスによって利用可否が異なります**。使えない場合は管理者に確認してください
- 会議の文字起こし・要約機能を使う場合、**参加者にその旨を事前に伝えることが望ましい**です
- **児童生徒・保護者の個人情報が含まれる会話の要約をCopilotで行うことは避けてください**
- 外部の参加者がいる会議では、Copilotの利用について追加の配慮が必要です

## 使う前に確認すべき学校・自治体のルール

Copilotを校務で使い始める前に、以下の点を必ず確認してください。**自己判断で利用を開始するのではなく、所属校・自治体のルールに従うことが大前提**です。

### 確認すべきポイント一覧

| 確認事項 | 確認先 |
|---|---|
| 生成AIの利用に関するガイドライン・方針の有無 | 教育委員会・管理職 |
| Microsoft 365 Copilotのライセンス導入状況 | ICT担当・管理者 |
| 校務端末での生成AI利用の可否 | ICT担当・管理職 |
| 個人アカウントでの利用可否 | ICT担当・管理職 |
| 入力してよいデータの範囲 | 情報セキュリティポリシー |
| 生成物の校外配布に関するルール | 管理職 |

### よくある自治体のパターン

自治体によって方針はさまざまです。以下は一般的に見られるパターンの例です（あくまで傾向であり、必ず自分の所属先を確認してください）。

- **積極的に活用を推進している自治体** — 職員向け研修を実施し、利用ガイドラインを策定している
- **条件付きで認めている自治体** — 個人情報を含まない範囲に限り、特定のサービスの利用を認めている
- **利用を制限している自治体** — セキュリティ上の理由から、当面の間は利用を制限している
- **明確な方針がまだない自治体** — 個別の判断に委ねられており、管理職との相談が必要

方針が明確でない場合は、まず管理職やICT担当に「校務で生成AIを試してみたいのですが、確認すべきルールはありますか?」と相談するところから始めてください。

### 端末の設定変更について

Copilotの利用にあたって端末の設定変更が必要な場合がありますが、**学校管理端末の設定変更を個人の判断で行うことは推奨しません**。設定変更が必要な場合は、必ずICT担当・管理者に依頼してください。

## 個人情報・セキュリティの注意点

校務でCopilotを使う際の最も重要なポイントは、**個人情報の取り扱い**です。これはCopilotに限らず、すべての生成AIサービスに共通する注意事項です。

### 入力してはいけない情報

以下の情報は、Copilotを含む生成AIサービスに入力しないでください。

| カテゴリ | 具体例 |
|---|---|
| **児童生徒の個人情報** | 氏名、学籍番号、出席番号と氏名の対応 |
| **成績・評価データ** | テスト点数、通知表評定、学力調査結果 |
| **要配慮個人情報** | 障害名、診断名、家庭環境、虐待・DV情報 |
| **保護者の個人情報** | 保護者名、連絡先、勤務先、家族構成 |
| **写真・動画** | 児童生徒が映っている画像・映像データ |
| **未公開の校内情報** | 人事情報、予算内部資料、懲戒関連の情報 |
| **アカウント情報** | パスワード、アクセスキー、認証情報 |

### Microsoft 365 Copilotの法人契約について

Microsoft 365 Copilot（法人向け）は、個人向けサービスとはデータの取り扱いが異なる場合があります。Microsoftは法人向けサービスについて、入力データを学習に使用しない旨を表明しています（ただし、規約や方針は変更される可能性があります）。

自治体・学校がどのプラン・契約形態で利用しているかによってデータの扱いが異なるため、**「うちの学校ではどうなっているか」をICT担当に確認することが重要**です。

### 安全に使うための工夫

個人情報を入力せずにCopilotを活用するための工夫をいくつか紹介します。

| やりたいこと | 安全な方法 |
|---|---|
| 所見の下書き | 氏名を入れず「算数が得意な児童」のように抽象化して依頼する |
| 保護者向け文書の作成 | 個人名・クラス名を入れず、一般的な内容で下書きを生成する |
| 成績データの分析 | 個人名を削除し、匿名化した仮データで関数やグラフの作り方を相談する |
| 会議メモの整理 | 個人が特定される発言内容を除いた要点のみを入力する |

「個人情報を入れてはいけないなら何もできない」ということはありません。**情報を抽象化・匿名化する工夫**によって、多くの業務でCopilotを安全に活用できます。

### パスワード・アカウント管理

- パスワードをCopilotやその他の生成AIに入力しない
- **アカウントの貸し借りは絶対に行わない**
- 他の教員のアカウントでCopilotを利用しない
- 共有端末を使用した後は必ずサインアウトする

### 外部へのデータ共有

- Copilotで生成した文書を外部に送る際は、意図しない情報が含まれていないか確認する
- **児童生徒の写真・動画・個人情報を外部のサービスやクラウドにアップロードすることは避けてください**
- ファイル共有の際は、共有範囲（校内のみ/外部にも公開）を必ず確認する

個人情報と生成AIの注意点について、さらに詳しくは[学校における生成AIの安全な使い方](/articles/school-generative-ai-privacy-security)をご確認ください。

## よくある誤解

Copilotについて、教員の間でよく見られる誤解を整理します。

### 誤解1：「CopilotはChatGPTと同じもの」

CopilotはMicrosoftが提供する独自のサービスです。AIモデルとしてOpenAI社の技術を活用していますが、サービスとしては別物です。利用規約、データの取り扱い、機能、料金体系はそれぞれ異なります。

### 誤解2：「Microsoft 365を使っていればCopilotも使える」

Web版Copilot（ブラウザ版）はMicrosoftアカウントがあれば基本的な機能を利用できますが、**Word・Excel・PowerPointなどのアプリ内で動作するMicrosoft 365 Copilotは、追加のライセンスが必要**です。学校で利用できるかどうかは、組織の契約状況によります。

### 誤解3：「Copilotなら個人情報を入力しても安全」

法人向けMicrosoft 365 Copilotではデータの扱いが個人向けとは異なるとされていますが、それでも**児童生徒の個人情報をAIに入力することは推奨しません**。セキュリティ上のリスクは技術的な問題だけでなく、倫理的・制度的な観点も含まれます。

### 誤解4：「Copilotが作った文章はそのまま使える」

Copilotを含むすべての生成AIは、**事実と異なる情報をもっともらしく出力する（ハルシネーション）ことがあります**。生成された文章・データ・数値は、必ず教員自身が確認・修正してから使用してください。特に、法令・制度・統計に関する記述は公式資料と照合が必要です。

### 誤解5：「Copilotで作った資料の著作権は自由に使える」

生成AIが作成した文章や画像の著作権については、法的な整理が進行中の部分が多く、**現時点で明確な結論が出ていない論点もあります**。商用利用や公的文書での使用にあたっては、最新の法的動向を確認し、慎重に判断してください。

### 誤解6：「全員が同じ環境で使える」

Copilotの利用可能な範囲は、**所属校・自治体・管理者の設定によって大きく異なります**。同じ自治体内でも学校ごとに設定が違う場合があります。「隣の学校で使えているからうちでも使えるはず」と思い込まず、自校の環境を確認してください。

### 誤解7：「AIを使えば仕事が全部楽になる」

Copilotは便利な支援ツールですが、**すべての業務が劇的に効率化されるわけではありません**。AIが得意な作業（文章のたたき台作成、定型文の生成、データ整理の補助など）と、AIには任せられない作業（教育的判断、児童生徒の観察に基づく評価、保護者対応の微妙なニュアンスなど）を区別することが重要です。

「AIを導入すれば残業がなくなる」という期待は現実的ではありません。**「AIで10分短縮できる作業を見つけて、その時間を子どもと向き合う時間に充てる」**という考え方が、学校現場に合った活用の姿勢です。

## まとめ

Microsoft Copilotは、Word・Excel・PowerPoint・Teamsといった学校現場で馴染みのあるアプリと連携できる生成AIサービスです。校務文書の作成・データ整理・会議の要約など、多くの業務で活用の可能性があります。

一方で、安全に活用するためには以下のポイントを必ず押さえてください。

### 活用の前提

- **所属校・自治体・管理者のルールを確認する**ことが最優先
- Microsoft 365 Copilotの**ライセンス導入状況**を確認する
- **個人情報は入力しない**（児童生徒の氏名・成績・家庭情報・写真など）
- 生成された内容は**必ず教員自身が確認・修正してから使用**する
- 端末の設定変更は**個人判断で行わず、管理者に相談**する

### 始め方のステップ

1. **ルール確認** — ICT担当・管理職に利用の可否と範囲を確認する
2. **小さく始める** — 個人情報を含まない一般的な文書作成から試す
3. **結果を確認する** — 生成された内容の正確性・適切性を必ずチェックする
4. **共有する** — うまくいった使い方を校内で共有し、ナレッジを蓄積する

生成AIは「仕事を代行する道具」ではなく、「教員の仕事を支援する道具」です。最終的な判断と責任は常に教員にあります。安全な範囲で少しずつ試しながら、ご自身の校務に合った活用法を見つけてください。

### うまく使うためのヒント

最後に、Copilotを校務で活用する際の実践的なヒントをいくつか挙げます。

- **プロンプトは具体的に書く** — 「文章を書いて」ではなく「小学4年生の保護者向けに、遠足のお知らせを300字で書いて」のように、対象・目的・分量を明示する
- **一度で完璧を求めない** — 最初の出力がいまひとつでも、「もう少しやわらかい表現にして」「箇条書きに変えて」と追加で指示すると改善される
- **校内で知見を共有する** — うまくいったプロンプトや活用法は、学年会や職員室で共有すると全体の効率が上がる
- **使わない判断も大切** — すべての業務にAIを使う必要はない。手書きの方が早い作業、対面で話した方がよい場面では、あえて使わない判断も重要

[AIで変わる校務改善](/articles/ai-koomu-kaizen-nyumon)の記事では、Copilotに限らない生成AI全般の校務活用を紹介しています。あわせてご覧ください。
