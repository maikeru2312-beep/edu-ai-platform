## はじめに：「AIで楽になる」はずが、増えていないか

「校務改善のためにAIを入れたはずなのに、なぜか楽になった実感がない」——そう感じている先生は少なくありません。ツールを増やすほど、使い方を覚える時間・確認する時間・周りに説明する時間が増えて、結局トータルの負担が減らない。これは、生成AIの導入でも起こりがちな落とし穴です。

教員の長時間労働は依然として解消されない課題です。その要因のひとつが、授業以外の校務——通知表、学年だより、会議、指導案作成——にかかる膨大な時間です。**ChatGPTをはじめとする生成AIがこの状況を変える可能性**として注目されていますが、問題は「使うかどうか」ではなく、「何のために、どこまで使うか」をはっきりさせられるかどうかにあります。

本記事では、**教員がAIを校務に活用する際の考え方・5つの活用例・任せてはいけない判断・始め方**を、現場で崩れやすいポイントとあわせて整理します。

> **教育DXナビの立場**
>
> 教育DXナビでは、AI活用を「使ったかどうか」ではなく、「子どもを見る時間・教材を整える時間・保護者と丁寧に関わる時間が、本当に取り戻せたか」で考えます。生成AIは、教師の判断を代行する道具ではありません。むしろ、教師の専門性が発揮される環境を整え、**先生が子どもを見る時間を取り戻すための道具**として使うべきものです。便利さだけで導入を急がず、ここからは「どこは任せてよく、どこは任せてはいけないか」を一緒に整理していきます。

> AIを使うかどうか・何に使うかの判断は、**所属校・教育委員会のガイドラインに従って**行ってください。本記事は一般的な活用可能性の紹介であり、特定のサービス利用を推奨するものではありません。

## 1. AIで校務改善できる理由

生成AIが校務効率化に向いている理由は、教員の校務に「パターンが決まっている文章作業」が多いからです。

- 所見・通知文・お便りは**書式・文体・文字数**がある程度決まっている
- 議事録・要点整理は**情報の構造化**が中心で、創造性より整理力が求められる
- 指導案・教材案のたたき台は**既存フォーマットへの当てはめ**が多い

このような作業は生成AIが得意とする領域です。**「AIが校務を代行する」のではなく、「AIが下書きを用意し、教員が仕上げる」という分業**によって、教員は最終判断・確認・修正に集中できます。

---

## 2. 教員の負担が大きい校務の具体例

文部科学省の調査でも明らかなように、教員の時間外業務の多くは授業外の事務・記録作業に費やされています。

**特に時間がかかる校務：**

| 業務 | 主な負担ポイント |
|---|---|
| 通知表所見 | クラス全員分を学期末に集中して記述 |
| 学年だより・保護者向け文書 | 月次・行事ごとに発行。文体・表現の質も求められる |
| 会議メモ・議事録 | 会議後に別途時間を割いて整理 |
| 指導案・教材案 | 単元ごとに一から作成。形式が複雑 |
| アンケート・自由記述の集計 | 記述式の回答を読み込んでまとめるのに時間がかかる |

これらの業務すべてに生成AIが使えるわけではありませんが、特に「文章を構造化する・下書きをつくる」部分ではAIの支援が有効です。

## 3. AIで効率化しやすい業務5つ

### ① 通知表所見の下書き

学期末に最も時間を取られる業務のひとつです。AIに**個人情報を含まない観点・エピソード・方向性**を入力すれば、下書き文を生成できます。

**プロンプト例：**

```
以下の観点をもとに、小学3年生の通知表所見（80字以内）を作成してください。

・観点：算数の授業態度と理解
・事実：計算問題に粘り強く取り組む。友達に教える場面も見られた
・方向性：意欲と協力の姿勢を前向きに表現する
```

**重要：** 氏名・学籍番号・診断名・家庭情報などの個人情報は入力しないでください。生成された文章は必ず確認・修正してから使用します。

→ 詳しくは[ChatGPTで通知表所見を作る方法と注意点【教員向け】](/articles/chatgpt-tsuchihyo-shoken)をご参照ください。

---

### ② 学級通信・保護者向け文書

月1回の学年だより、行事のお知らせ、配布物の文章など、個人情報を含まない**全体向け文書**はAIで大幅に効率化できます。

**プロンプト例：**

```
6月の学年だより（小学2年生向け）の本文を作成してください。
テーマ：梅雨の季節の生活習慣と、プール開始のお知らせ
トーン：保護者向けに丁寧かつ明るく
文字数：400字程度
```

下書きをAIに生成させ、学校独自の情報や表現を加えて完成させます。一から書くより大幅に短縮できます。

---

### ③ 会議メモ・議事録の整理

職員会議や学年会の後、要点をまとめるのに時間がかかる場合があります。**個人名・内部情報を除いた箇条書きメモ**をAIに渡して整理させることで、議事録の構造化を補助できます。

**プロンプト例：**

```
以下の会議メモから、議事録形式（決定事項・確認事項・次回検討事項）に整理してください。

メモ：
・運動会の日程について、10月15日に決定
・プログラム案を各学年が持ち寄ること
・当日の保護者席については引き続き検討
・熱中症対策として水分補給の時間を設ける
```

**注意：** 未公表の方針・予算・人事に関する情報は入力しないでください。

---

### ④ 指導案・教材案のたたき台

単元名・学年・目標を入力するだけで、指導案の骨格をAIに生成させることができます。

**プロンプト例：**

```
小学4年生の理科「季節と生き物（夏）」の単元で、
1時間の授業指導案（目標・主な学習活動・評価規準）の骨格を作成してください。
```

AIが生成した骨格は、教員が内容の正確性・学習指導要領との整合・実態への適合を確認した上で修正・活用します。

---

### ⑤ アンケート・自由記述の整理

保護者アンケートや学校評価の自由記述を整理する際、**個人が特定されない形に加工した内容**であれば、AIで分類・要点抽出を補助させることができます。

**活用例：**

```
以下の意見をポジティブ・要望・その他に分類し、
各カテゴリの主な傾向を3点ずつまとめてください。

意見一覧：（個人情報を除いたテキストのみ貼り付ける）
```

**注意：** 氏名・連絡先・個人が特定できる記述が含まれている場合は、必ず除いてから入力してください。

## 4. 現場でよくある失敗例

便利な道具ほど、使い方を間違えると逆に時間を奪います。導入の前に、次のような「あるある」を知っておくと、同じつまずきを避けやすくなります。

- **下書きを直す時間のほうが長くなった** — 実態と合わない一般的な文章が出てきて、結局ゼロから書き直した。観点や事実を具体的に渡せていないと起こりがちです。
- **AIで作った所見が、その子の姿と合っていなかった** — もっともらしい文面ほど、確認が甘くなりやすい。便利さが、子どもを見て書くという本来の作業を飛ばしてしまうことがあります。
- **入力する人だけが疲弊した** — 情報共有ツールを入れたが、入力ルールが曖昧で、結局一部の先生だけが入力役を担い、負担が偏った。
- **ICT担当だけが頑張り、学校全体に広がらなかった** — 「詳しい人が代わりにやる」状態になり、属人化して持続しなかった。
- **「便利そうだから」で始めて、目的が後から消えた** — 何の時間を取り戻したかったのかが曖昧なまま使い続け、振り返ると負担は減っていなかった。

ここで大切なのは、これらは「AIが悪い」のではなく、**目的と任せる範囲の設計が抜けていた**という点です。順番を間違えると現場で崩れやすいのは、「ツールを決めてから使い道を探す」進め方です。逆に「取り戻したい時間を決めてから道具を選ぶ」と、失敗はぐっと減ります。

### 現場で考えるミニケース

ある学年で、学年だよりの作成を生成AIで効率化することにしました。最初の月は下書きが一瞬ででき、確かに時短になりました。ところが数か月後、「AIっぽい無難な文章」が続き、保護者に伝えたい学年の様子や行事の温度感が薄れていることに気づきました。

この場合、AIをやめる必要はありません。**事実とねらい（今月いちばん伝えたいこと）を先に箇条書きで決めてからAIに渡す**、最後の一段落だけは担任の言葉で書く、と決めるだけで、時短と「らしさ」は両立できます。

このように、AI校務改善では「使うか使わないか」よりも、**「何を伝えたいから・何の時間を取り戻したいから使うのか」を先に決める**ことが、長続きする使い方の分かれ目になります。

## 5. AIに任せてはいけない業務

効率化の一方で、AIに頼ってはいけない判断・業務があります。

| 業務・判断 | 理由 |
|---|---|
| 児童生徒の評価・成績の決定 | 専門的・倫理的判断が必要 |
| 保護者との重要な個別連絡の内容 | 文脈・信頼関係が必要。誤解が生じやすい |
| 特別支援に関する個別の対応方針 | 専門性・関係機関との連携が前提 |
| 緊急対応・危機管理の判断 | リアルタイムの状況判断が必要 |
| 教員としての説明責任が必要な文書 | 最終責任は教員にある |

これらはAIを補助的に使うことはできても、**最終判断は必ず人間（教員・管理職）が行う**という原則を守ってください。

## 6. 個人情報・セキュリティ上の注意

生成AIを校務に使う上で、**絶対に守ってほしい原則**が個人情報の保護です。

### 入力してはいけない情報

- 児童生徒の氏名・学籍番号・成績
- 障害名・診断名・家庭状況などの要配慮個人情報
- 写真・動画・音声データ
- 学校内部の未公開情報（会議内容・予算・人事など）

### なぜ問題になるか

生成AIサービスは入力データが外部サーバーへ送信されます。サービスの利用規約によっては入力内容がAIの学習に使われる可能性があり、個人情報保護法や校内規定との兼ね合いが生じます。

### 安全に使う工夫

- 個人名を「担当児童」「この子」などに置き換える（**匿名化**）
- 具体エピソードを特定できない程度に一般化する（**抽象化**）
- 実例の代わりに架空の設定で入力する（**架空例への置き換え**）

→ 詳しくは[学校で生成AIを使うときの個人情報・セキュリティの注意点](/articles/school-generative-ai-privacy-security)をご参照ください。

## 7. 校内ルール・教育委員会方針の確認

生成AIの利用に関するルールは、**教育委員会・学校によって異なります**。

- 生成AIの業務利用を認めているか
- 個人情報の外部サービス入力の可否
- 学校支給端末での外部AIサービス利用の可否
- 管理職への事前報告・承認フローの有無

方針が明示されていない場合も、「ルールがないから自由」ではなく、**ICT担当・情報主任・管理職に確認してから**使い始めることが大切です。ガイドラインが整備されていない学校では、相談が校内ルール整備のきっかけになることもあります。

## 8. 初心者教員向けの始め方

「AIを使ってみたいけど何から始めればいいかわからない」という方は、次の3ステップで始めることをお勧めします。

### Step 1：個人情報を含まない業務から試す

最初は**全体向け文書（お便り・通知文）や教材アイデア出し**から始めましょう。個人情報を含まないため、リスクが低く試しやすい業務です。

### Step 2：校内ルールを確認する

AIツールを使い始める前に、**ICT担当または管理職に一言相談**しましょう。「〇〇を使って通知文の下書きを作りたいのですが、問題ありませんか?」という確認だけで十分です。

### Step 3：出力を「たたき台」として使う

AIの出力はあくまで出発点です。内容が正確かどうか、表現が学校の方針に合っているかを確認し、**必ず教員の言葉で仕上げてから使います**。この習慣がAI活用の基本です。

**使いやすいツールの例（参考）：**

- ChatGPT（OpenAI）— 個人・学校双方で利用実績が多い
- Gemini（Google）— Google Workspace for Education との親和性が高い
- Microsoft Copilot — Microsoft 365 と連携

いずれも無料プランと有料プランがあり、学校での利用ルールはサービスごとに異なります。使用前に規約と校内方針の両方を確認してください。

## 9. 公式情報をどう読むか

AI校務改善を進めるときは、次の一次情報を「現場でどう読むか」とあわせて押さえておくと、判断がぶれません。

- **文部科学省「初等中等教育段階における生成AIの利用に関するガイドライン」** — 生成AIを「使う／使わない」の二択ではなく、教育活動の目的に照らして判断する姿勢が示されています。現場では、まず「自校が利用を認めているか」「個人情報の入力可否」をこのガイドラインと校内規定の両方で確認するのが出発点です。
- **個人情報保護委員会の解説（要配慮個人情報）** — 障害・診断・健康に関する情報は「要配慮個人情報」にあたり、取り扱いに特に注意が必要です。現場では、これらを生成AIに入力しないという一線を引く根拠になります。
- **各サービスの利用規約・データの取り扱い（OpenAI／Google／Microsoft など）** — 入力データが学習に使われるか、保存期間はどうか、教育向けプランで扱いが変わるかは、サービスごと・プランごとに異なります。「無料だから安全」ではなく、規約を確認してから業務利用に進んでください。

公式情報は、貼って終わりにせず「自校のルールに照らして何を確認するか」までセットで読むことが大切です。最新の内容は必ず各機関・各社の公式ページでご確認ください。

## 明日から確認できるチェックリスト

AIを校務に使う前後で、次の点をざっと確認してみてください。

- [ ] 氏名・診断名・成績など、個人情報や要配慮個人情報を入力していないか
- [ ] 目的が「便利そうだから」だけになっていないか（何の時間を取り戻したいか言えるか）
- [ ] 出力をそのまま使わず、教員の言葉で確認・修正しているか
- [ ] その文章は、目の前の子どもや学年の実態と合っているか
- [ ] 入力する人だけに負担が偏っていないか
- [ ] 校内ルール・教育委員会の方針と矛盾していないか
- [ ] 保護者への説明が必要な内容を、AI任せにしていないか

ひとつでも引っかかったら、使い方を少し設計し直す合図です。

## まとめ

生成AIを校務に活用することで、**下書き・整理・発想の部分を大幅に短縮し、教員が本来の仕事——子どもと向き合う時間や専門的判断——に集中できる**環境を作ることができます。

**活用できる5業務のおさらい：**

1. **通知表所見の下書き** — 個人情報を除いた観点・方向性を入力
2. **学年だより・保護者向け文書** — 個人情報を含まない全体向け文書
3. **会議メモ・議事録の整理** — 箇条書きを構造化
4. **指導案・教材案のたたき台** — 骨格を生成し、教員が仕上げる
5. **アンケート自由記述の整理** — 個人情報を除いた内容を分類・要点化

**守るべき3原則：**

1. 個人情報はAIに入力しない
2. 出力はそのまま使わず、必ず確認・修正する
3. 校内ルールを確認してから使い始める

「AIに任せる」のではなく「AIを使いこなす」姿勢が、長続きするAI活用の基本です。まずは個人情報を含まない一業務から試してみましょう。
