## はじめに

学期末のたびに多くの時間を費やす業務のひとつが、通知表の所見作成です。クラス全員分を丁寧に書き上げるのは、授業準備や行事対応と並行するなかで大きな負担になります。

近年、**ChatGPTなどの生成AIを使って所見の下書きを作成する教員が増えています**。うまく活用すれば作業時間の短縮につながりますが、「個人情報の入力」「そのまま使ってしまう」といった使い方には重大なリスクがあります。

本記事では、安全で効果的なChatGPT活用の考え方・手順・プロンプト例・注意点を、教員の立場からわかりやすく解説します。

> 本記事は一般的な活用方法の紹介です。AIツールの利用については、**所属校・教育委員会のガイドライン・方針を必ず確認の上**、自己判断・自己責任で行ってください。

---

## 1. ChatGPTで所見作成を効率化できる理由

通知表所見は、一定のパターンと文体を持つ文章です。

- 観点（学習・行動・成長など）ごとの記述
- 前向きな言葉づかいと丁寧な文体
- 60〜120字程度の決まった文字数

このように**構造が決まっている文章の下書き生成**は、生成AIが特に得意とする作業です。教員が観点やキーワードを入力すれば、下書きとなる文章を瞬時に出力してくれます。

ただし、**AIはあくまで下書きを補助するツール**です。最終的な内容の判断・修正・確認は必ず教員が行います。「AIが所見を書く」のではなく、「教員がAIの助けを借りて所見を仕上げる」という位置付けが正しい理解です。

---

## 2. 通知表所見作成でAIを使うときの基本姿勢

所見作成にAIを使う前に、以下の姿勢を確認してください。

**✅ やること**
- 入力前に個人情報を取り除く（匿名化・抽象化）
- 出力された文章を必ず読んで確認・修正する
- 事実と異なる表現が含まれていないか照合する
- 所属校・教育委員会の方針を事前に確認する

**❌ やらないこと**
- 児童生徒の氏名・学籍番号をAIに入力する
- 病名・障害名・家庭環境などの要配慮情報を入力する
- 生成された文章をそのままコピーして使用する
- 学校支給端末以外の環境でのAI利用（ルール次第）

この2つを守るだけで、リスクを大幅に下げることができます。

## 3. 児童生徒の個人情報をそのまま入力してはいけない理由

ChatGPTをはじめとするAIサービスは、入力した内容がサービス提供企業のサーバーに送信されます。

**個人情報を入力することによるリスク：**

1. **情報漏洩リスク** — 入力データがどのように管理・利用されるかはサービスごとに異なります
2. **個人情報保護法への抵触** — 要配慮個人情報（障害・病歴など）の第三者提供には本人同意が必要です
3. **校内規定・教育委員会方針への違反** — 多くの教育委員会は個人情報の外部サービス入力を禁じています
4. **保護者・保護者からの信頼失墜** — 万が一の情報漏洩は保護者・地域の信頼に直結します

「無料ツールだから大丈夫」「匿名化しているつもり」という判断ではなく、**所属校の情報主任・管理職に確認してから使い始める**ことが基本です。

## 4. 安全な使い方：4つの原則

### 原則① 匿名化する

入力する前に、児童生徒が特定できる情報をすべて取り除きます。

| 入力してはいけない情報 | 代わりに使う表現 |
|---|---|
| ○○さん（氏名） | 「この子ども」「担当児童」 |
| A組の男子 | 「小学○年生の児童」 |
| 自閉スペクトラム症 | 「コミュニケーションに支援が必要な児童」 |
| 家庭が複雑な子 | （入力しない） |

### 原則② 観点・事実だけを入力する

「この子はこういう子」という評価ではなく、**観点・行動・成長の事実**だけを入力します。

**例：**
> × 「消極的でなかなか発言できない子です」  
> ○ 「授業中の挙手発言が少ないが、グループ活動では積極的に意見を伝える場面が見られた」

### 原則③ 事実情報を抽象化する

具体的なエピソードも、特定の児童が識別できない程度に抽象化します。

> × 「運動会のリレーで転んでも最後まで走りきった」  
> ○ 「困難な場面でも最後まで粘り強く取り組んだ」

### 原則④ 最終判断は必ず教員が行う

生成された文章は**必ず読み直し、事実と照合**します。AIは文章として自然な表現を生成しますが、「その子の実態に合っているか」の判断は教員にしかできません。

## 5. プロンプト例

### ケース①：よい点を膨らませたい

```
以下の観点をもとに、小学校通知表の所見文（80字以内）を作成してください。

・観点：算数の授業態度と理解
・事実：計算問題に粘り強く取り組む。友達に教える場面も見られた
・方向性：意欲と協力の姿勢を前向きに表現する
```

### ケース②：課題を前向きに表現したい

```
以下の観点をもとに、通知表所見（80字以内）を作成してください。
課題を直接指摘せず、「次の目標」や「成長の途中」として前向きに表現してください。

・観点：国語の書く活動
・事実：文章を書くことへの苦手意識がある。短い文は書ける
・方向性：できていることを認め、次のステップを示す
```

### ケース③：短いメモから所見文を作りたい

```
以下のメモをもとに、保護者に伝わる通知表所見（80字以内）を作成してください。
主語は「〇〇さん」ではなく「友達と」「学習では」のように書き出してください。

メモ：体育得意・給食委員で活躍・授業は静かに参加・係活動まじめ
```

### ケース④：特別支援教育での配慮ある表現に直したい

```
以下の所見文を、特別支援教育における配慮表現に修正してください。
「できない」「苦手」という表現を使わず、成長・支援・次の目標につながる言葉にしてください。

修正前：「漢字の書き取りが苦手で、テストでミスが多い」
```

**出力例のイメージ：**
> 「漢字の練習に丁寧に取り組んでいます。書き順を意識する場面が増えてきており、引き続き繰り返し練習を通じて定着を図っていきます。」

## 6. 出力された文章をそのまま使わない理由

AIの生成する文章には、次のような問題が含まれる場合があります。

| 問題 | 具体例 |
|---|---|
| **事実と異なる内容** | 実際には見られていない行動が書かれる |
| **過度な称賛** | 実態以上にポジティブな評価になる |
| **画一的な表現** | クラス全員の所見が似通った文体になる |
| **保護者に違和感を与える表現** | 機械的・他人行儀な文体になる場合がある |
| **ハルシネーション** | もっともらしいが実際にはない事実を生成する |

所見は子どもの学びや成長を保護者に伝える大切な文章です。AIの出力をそのまま使うのではなく、**教員の言葉でひと手間加えることが信頼性を守ります**。

## 7. 管理職・校内ルール・教育委員会方針の確認

生成AIの教育利用は、2024〜2026年現在も各教育委員会や学校によって方針が異なります。

**確認すべき主な事項：**

- 学校支給端末での外部AIサービス利用の可否
- 個人情報・要配慮個人情報の外部サービス入力の可否
- 職員個人のアカウントでの利用可否
- 管理職や情報主任への事前報告・承認フローの有無

「他の先生も使っているから大丈夫」という判断は禁物です。**使い始める前に管理職または情報主任に確認し、記録に残しておく**ことを強くお勧めします。

## 8. よくある失敗例

### 失敗例① 氏名をそのまま入力した

> 「○○くんの所見を書いて」と入力してしまった

→ 外部サービスへの個人情報送信になります。氏名は必ず取り除いてください。

### 失敗例② 生成文をクラス全員に流用した

> 同じプロンプトで生成した文章を、表現を少し変えて全員に使った

→ 保護者が読んで不自然に感じたり、他の家庭と似た所見であることに気づかれることがあります。必ずその子どものエピソードを反映して修正を加えてください。

### 失敗例③ 病名・診断名を入力した

> 「ADHD傾向のある児童の所見を」と入力した

→ 診断名・障害名は要配慮個人情報です。診断名を使わず「集中を保つことが難しい場面がある児童」など行動の事実で入力してください。

### 失敗例④ 確認せずに印刷した

> 生成された文章を確認せずにそのまま通知表に貼り付けた

→ AIは事実と異なる内容を自信満々に出力することがあります（ハルシネーション）。必ず一字一句確認してから使用してください。

## まとめ

ChatGPTによる通知表所見の下書き作成は、**正しく使えば時間短縮と質の向上に有効なツール**です。ただし、次の点は絶対に守ってください。

1. **個人情報（氏名・診断名・家庭情報）は入力しない**
2. **生成された文章はそのまま使わず、必ず確認・修正する**
3. **所属校・教育委員会の方針を事前に確認する**
4. **最終判断は常に教員が行う**

「AIに所見を書いてもらう」のではなく、「AIを使って所見を書く時間を短縮し、子どもと向き合う時間を確保する」という視点で活用しましょう。
