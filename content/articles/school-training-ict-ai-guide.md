GIGAスクール構想により1人1台端末が整備され、生成AIの校務活用も広がりつつあります。しかし「端末はあるが使い方がわからない」「AIの話題は出るが、何をどこまでやってよいか不安」という声は、現場の先生方から今も多く聞かれます。

こうした状況で鍵を握るのが**校内研修**です。外部の講演を聞くだけでなく、自分たちの学校・授業・校務に合った形で学び合い、日常の実践につなげていく——そのための研修を校内でどう設計し、どう進めればよいかを本記事で整理します。

> 本記事は一般的な研修設計の考え方と進め方の整理です。個別の法的・行政的判断を代行するものではありません。実際の研修内容・ツール選定・利用範囲については、**所属校・教育委員会のガイドライン・方針を必ず確認の上**進めてください。

## 校内研修が定着しにくい理由

校内研修でICTやAI活用を扱っても、「聞いたときは納得したが、翌日から使えていない」「一部の教員しか参加しなかった」という経験をもつ方は少なくないのではないでしょうか。定着しにくい背景には、いくつかの構造的な原因があります。

### 時間が確保しにくい

放課後の限られた時間に全教員を集めること自体が難しく、研修が短時間で終わる「説明だけ」の場になりがちです。じっくり操作を試す時間が取れないまま終わってしまいます。また、学期末や行事前は特に余裕がなくなり、「研修よりも目の前の業務を優先したい」というのが正直な気持ちである場合も多いでしょう。

### テーマが遠い

「教育DXの最新動向」のような大きなテーマは、目の前の授業や校務との接点が見えにくいことがあります。参加した教員が「自分の仕事にどう活かせるか」をイメージできないと、実践にはつながりません。研修テーマは「明日の授業や来週の校務に役立つか」という視点で選ぶことが重要です。

### 「できる人」と「できない人」の差

ICTスキルや関心に大きな差がある中で、同じ内容を一斉に扱うと、得意な人には物足りなく、苦手な人にはついていけない研修になってしまいます。特に、得意な教員が「こんなの簡単ですよ」と発言すると、苦手な教員はますます発言しにくくなり、次回以降の参加意欲も下がるという悪循環が生まれることがあります。

### 一度きりで終わる

研修が「聞いて終わり」の単発イベントになると、日常の授業や校務に結びつく前に記憶が薄れます。**研修後のフォローアップや実践の場をセットで設計すること**が、定着の分かれ目です。

→ 教育DX全体の考え方については[教育DXとは何か？現場教員が押さえておきたい基礎知識](/articles/kyoiku-dx-kiso)もご参照ください。

## ICT・AI研修で最初に決めること

研修を企画するとき、テーマや内容の前にまず決めておきたいことがあります。

### 目的を1つに絞る

「ICTを広めたい」は目的としては広すぎます。次のように、**1回の研修で扱う目的を1つに絞る**と、参加者が「何のための時間か」を理解しやすくなります。

- 授業での端末活用を1つ増やす
- 生成AIで校務文書の下書きを体験する
- 校内のICTルールを全員で共有する

### 対象を明確にする

全教員向けか、希望者向けか、特定の学年・教科・分掌のメンバー向けかで、内容の深さや進め方が変わります。無理に全員を同じ場に集めず、小グループで複数回に分ける方が効果的な場合もあります。

### 役割分担を決める

研修を1人のICT担当にすべて任せると、負担が集中して続きません。次のような役割を分けることで、持続可能な体制になります。

| 役割 | 担当例 | 主な仕事 |
|---|---|---|
| 企画・全体設計 | 管理職・研修主任 | テーマ設定・日程調整・校内方針との整合 |
| 内容準備・進行 | ICT担当・情報主任 | 研修資料作成・当日のファシリテーション |
| 実践の橋渡し | 学年主任・教科主任 | 研修内容を日常の授業・校務に落とし込む声かけ |
| サポート | ICTに慣れた教員 | 当日の操作補助・研修後の個別フォロー |

管理職の理解と関与は特に重要です。管理職が「この研修は学校として必要だ」と位置づけることで、参加率や真剣度が変わります。

## 研修テーマの選び方

テーマ選びは「最新のトレンドを扱う」ことよりも、**「今、この学校で教員が困っていること・知りたいことに応える」**ことが優先です。

### 授業改善につながるテーマ例

- タブレット端末を使った授業での提示・共有の基本操作
- 学習支援アプリの基本的な使い方
- 児童生徒のタイピング指導・端末管理のコツ

→ タブレット活用の具体例は[タブレット1台で始める授業ICT活用](/articles/tablet-ict-jugyo-giga)も参考になります。

### 校務効率化につながるテーマ例

- 校務支援システムの便利な機能（使い切れていない機能の確認）
- 生成AIで文書の下書きを作る体験
- クラウドでの資料共有・同時編集の基本

### 安全面・ルール理解のテーマ例

- 個人情報の取り扱いと生成AIに入力してはいけない情報
- 情報モラル教育の授業設計
- 校内情報セキュリティルールの確認

→ 情報モラル教育については[情報モラル教育の進め方](/articles/information-morals-education-themes)で詳しく解説しています。

### テーマ設定の判断軸

迷ったときは、次の3つの観点で優先順位をつけると判断しやすくなります。

1. **緊急度** — 今すぐ全員が知る必要があるか（セキュリティルール変更など）
2. **影響範囲** — 多くの教員の日常業務に関わるか
3. **実践のしやすさ** — 研修直後から試せる内容か

## 研修前アンケートで把握すること

研修の効果を高めるために、事前に簡単なアンケートを実施することをお勧めします。Google フォームやMicrosoft Formsなど、校内で利用可能なツールで3〜5問程度の短いものが適しています。

### 聞いておきたい項目

| 項目 | 目的 |
|---|---|
| 現在のICT活用頻度 | 参加者のレベル感を把握する |
| 困っていること・知りたいこと | テーマの微調整に反映する |
| 生成AI利用経験の有無 | 研修のスタートラインを決める |
| 研修に期待すること | 参加者の関心を事前に知る |
| 希望する研修形式（講義型・体験型など） | 進め方の設計に反映する |

アンケート結果を研修の冒頭で簡単に共有すると、「自分たちの声が反映されている」という納得感につながります。

### アンケート実施時の注意

- **回答は無記名でも構いません**。ICTスキルに関する質問は、記名式だと正直に回答しにくい場合があります
- 「苦手」「使っていない」という回答が出ることを前提に設計してください。苦手であること自体を問題視するアンケートにならないよう配慮が必要です

## 研修当日の進め方

### 基本の構成（60分の場合）

校内研修は60分程度で組まれることが多いため、以下は60分の場合の進行例です。

| 時間 | 内容 | ポイント |
|---|---|---|
| 0〜5分 | 導入：本日のゴールと流れの説明 | 「今日の研修後に○○ができるようになる」と具体的に伝える |
| 5〜15分 | 説明：テーマの背景と基本知識 | スライドは最小限。必要な情報だけ簡潔に |
| 15〜40分 | 体験：実際に手を動かす時間 | 全体の半分以上を体験に充てる。個別サポートを手厚く |
| 40〜50分 | 共有：体験した感想・気づきの交流 | ペアやグループで短く共有。全体発表は任意 |
| 50〜60分 | まとめ：次のアクション・ミニ課題の提示 | 「明日から1つだけ試してみましょう」で締める |

### 進行のポイント

**体験時間を削らない**ことが最も重要です。説明に時間を使いすぎて「聞いただけ」で終わることが、研修が定着しない最大の原因です。説明は10分以内に抑え、残りは手を動かす時間に充てます。

**操作手順は紙またはデジタルで配布**してください。画面だけで説明すると、操作に手間取った教員が追いつけなくなります。手元で確認できる手順書があると安心感が生まれます。

**質問しやすい空気**をつくることも大切です。「わからないことは恥ずかしいことではない」という前提を冒頭で伝え、ICTが得意な教員が隣で個別にサポートする体制をとると、参加者の安心感が高まります。

### 30分しか取れない場合

行事の合間などで60分が確保できない場合は、以下のようにコンパクトにまとめることも可能です。

| 時間 | 内容 |
|---|---|
| 0〜3分 | ゴールと流れの説明 |
| 3〜8分 | 要点の説明（ポイントを3つに絞る） |
| 8〜23分 | 体験（1つの操作に集中する） |
| 23〜28分 | ペアで感想共有 |
| 28〜30分 | ミニ課題の提示 |

短時間でも「体験＋ミニ課題」の構造を維持することが、実践につなげるための最低条件です。

## 実践につなげるミニ課題

研修を「聞いて終わり」にしないために、研修後に取り組む**ミニ課題**を設定することが有効です。大がかりな課題ではなく、日常の業務や授業の中で「1つだけ試してみる」という小さなアクションが定着の起点になります。

### ミニ課題の例

| 研修テーマ | ミニ課題 |
|---|---|
| タブレット活用（授業） | 次の1週間で、1回だけ画面提示に端末を使う |
| 生成AIの基本操作 | 架空の学級通信の下書きをAIで1回作ってみる |
| クラウド共有 | 学年共有フォルダに1つファイルをアップロードする |
| 情報セキュリティ | 自分のパスワードの管理状況を確認する |

### ミニ課題設計のコツ

- **所要時間10分以内**で完結する内容にする
- **個人情報を扱わない**安全な題材を指定する
- **正解がない**課題にする（「やってみた」こと自体が成果）
- **報告の場を設定する**（次の職員会議や校内チャットで共有）

ミニ課題の結果を共有する場があることで、「やってみよう」というモチベーションが生まれます。共有の場では「うまくいった話」だけでなく「うまくいかなかった話」も歓迎すると、次の研修のヒントになります。

### ミニ課題と日常業務の接点

ミニ課題はできるだけ「どうせやる業務」と重ねるのが理想です。例えば、学級通信を書く予定がある教員にはAIで下書きを試すミニ課題を、来週授業参観がある教員にはタブレットで画面提示を試すミニ課題を割り当てるなど、**日常の予定と接続させる**と自然に取り組めます。

追加の業務として課すのではなく、「いつもの仕事を少し違うやり方で試してみる」という位置づけにすることがポイントです。

## 初心者教員への配慮

ICT・AI研修では、参加者のスキル差が大きな課題です。重要なのは、**ICTが苦手な教員を責めない・追い詰めない**研修設計を意識することです。

### 「できない」ことは問題ではない

ICTスキルは経験と環境に左右されるものであり、スキルが低いことは教員としての能力とは別の問題です。研修の冒頭で「今日はゼロからで大丈夫です」と明確に伝えてください。

### 習熟度別の配慮

| レベル | 配慮のポイント |
|---|---|
| 初心者 | 操作手順を1ステップずつ示す。隣にサポーターを配置する |
| 中級者 | 基本を確認しつつ、応用の選択肢を提示する |
| 上級者 | サポーター役やファシリテーターとして活躍してもらう |

上級者をサポーター役にすることで、上級者自身にとっても「教えることで学ぶ」機会になります。ただし、**特定の教員にサポート役を押し付ける構造にならないよう**、管理職や研修担当が配慮してください。

### 避けたい言葉と姿勢

研修の場では、次のような表現を避けるよう進行役が意識しておくとよいでしょう。

- 「まだやっていないのですか」「このくらいは知っていますよね」
- 「若い先生ならできるでしょう」（年齢とスキルは関係ありません）
- 「こんな簡単なこと」「基本中の基本」

ICTに慣れていない教員が「質問したら恥ずかしい」と感じてしまうと、研修の効果はゼロになります。**全員が安心して参加できる場づくり**が、研修設計で最も大切なことのひとつです。

### 個別サポートの仕組み

研修当日だけでなく、研修後にも個別に質問できる窓口があると、初心者教員の安心感が持続します。

- **ICT相談タイム** — 週1回、放課後に15分間、ICT担当が職員室で質問を受ける時間を設ける
- **ペア制度** — ICTに慣れた教員と苦手な教員をゆるくペアにし、困ったときに声をかけやすくする
- **校内FAQ** — よくある質問と操作手順を共有フォルダにまとめておく

いずれも大がかりな仕組みではなく、**小さく始めて続けること**が重要です。

## 生成AI研修で必ず扱う安全面

生成AIの研修では、操作方法や活用アイデアだけでなく、**安全面のルールを明確に扱う**ことが不可欠です。便利さだけを伝えて安全面を省略すると、後からトラブルにつながる危険があります。

### 入力してはいけない情報

生成AIサービスは、入力した情報がサービス提供会社のサーバーに送信されます。以下の情報は原則として入力しないことを研修で明確に伝えてください。

- **児童生徒の氏名・学籍番号・成績データ**
- **障害名・診断名・家庭環境などの要配慮個人情報**
- **写真・動画・音声データ**
- **教職員の人事情報・未公開の校内情報**
- **保護者との個別のやり取り内容**

これらの情報を生成AIに入力することは、個人情報の外部送信にあたる可能性があり、情報漏洩のリスクが発生します。

→ 詳しくは[学校における生成AIの安全な使い方](/articles/school-generative-ai-privacy-security)で整理しています。

### 出力の確認は教員の責任

生成AIはもっともらしい文章を生成しますが、事実と異なる内容（ハルシネーション）を出力することがあります。**AIの出力をそのまま児童生徒・保護者に提示しない**こと、必ず教員が内容を確認・修正してから使うことを研修で伝えてください。

研修の体験時間に「わざと間違った情報が含まれる出力」を見せて、参加者に間違いを見つけてもらうワークを取り入れると、ハルシネーションの実感を持ちやすくなります。

### 著作権について

生成AIの出力物の著作権については、現時点で法的な解釈が確定していない部分もあります。研修では「AIが生成した文章や画像の著作権については議論が進行中であり、**そのまま転載・配布する際は注意が必要**」という程度にとどめ、過度に断定しないことが適切です。

### アカウント管理

- **パスワードの共有やアカウントの貸し借りはしない**
- 学校が契約・指定しているサービスがある場合は、そちらを優先して利用する
- 個人アカウントで校務データを扱ってよいかは、所属校・教育委員会の方針を確認する

### 学校管理端末での注意

学校が管理している端末の設定変更（ブラウザ拡張機能の追加・ソフトウェアのインストールなど）は、**個人の判断で行わず、必ずICT担当や管理職に相談**してください。管理端末のセキュリティ設定を個人が変更すると、校内ネットワーク全体に影響を及ぼす場合があります。

→ ChatGPTの基本的な使い方は[教員のためのChatGPT入門](/articles/chatgpt-teacher-beginner-guide)、Microsoft Copilotについては[教員向けMicrosoft Copilot活用ガイド](/articles/microsoft-copilot-teacher-guide)で解説しています。

## 研修後の共有と振り返り

研修の効果は、当日ではなく**研修後の1〜2週間**で決まります。学んだことを実践に移し、その経験を共有する仕組みをあらかじめ設計しておくことが重要です。

### 共有の方法

| 方法 | 特徴 |
|---|---|
| 職員会議での短時間報告 | 全員に届く。1〜2分×数名程度で負担が少ない |
| 校内チャット・掲示板 | 非同期で手軽に投稿できる。写真やスクリーンショットも共有しやすい |
| 学年会・教科会での情報交換 | テーマに関連する教員同士で具体的な話ができる |
| ミニ展示（職員室にポスター掲示） | 研修に参加できなかった教員にも内容が伝わる |

### 振り返りシートの活用

研修直後に短い振り返りシートに記入してもらうと、研修の改善に役立ちます。

- 今日の研修で一番役に立った内容は何ですか
- 明日から試してみたいことは何ですか
- わからなかったこと・もっと知りたいことはありますか
- 研修の進め方について改善してほしい点はありますか

振り返りシートは**3〜4問、選択式と自由記述の組み合わせ**で、5分以内に回答できる分量が適切です。

### フォローアップの場を設ける

1回の研修で完結させず、2〜4週間後に短いフォローアップの場を設けると効果が高まります。フォローアップは15〜30分程度で十分です。

- ミニ課題の結果を共有する
- 実践してみて困ったことを出し合う
- 次のステップを確認する

**「やりっぱなし」にしない仕組みをあらかじめ研修計画に組み込む**ことが、定着の最大のポイントです。

### 管理職の関わり方

フォローアップの場では、管理職が参加して「先生方の取り組みを認める」言葉を添えることで、教員のモチベーションが大きく変わります。評価や監視ではなく、**応援と承認の姿勢**で関わることが大切です。

「うまくいかなかった」という報告に対しても、「試したこと自体が前進です」とフィードバックすることで、次の挑戦へのハードルが下がります。

## 校内ルールとの接続

ICT・AI研修の内容は、**校内の既存ルールと矛盾しないこと**が大前提です。研修で「こう使いましょう」と伝えた内容が校内ルールと食い違うと、現場が混乱します。

### 確認すべき校内ルール

- **情報セキュリティポリシー** — 端末の持ち出し・データの保存場所・パスワード管理
- **生成AI利用に関するガイドライン** — 利用可能なサービス・入力禁止事項・利用記録の要否
- **個人情報保護に関する規定** — 児童生徒の情報の取り扱い・外部サービスへの送信可否
- **端末利用ルール** — インストール可能なアプリ・設定変更の権限

### ルールが整備されていない場合

生成AIの利用ルールがまだ校内で整備されていない場合は、研修がルール策定のきっかけになることもあります。ただし、**ルールが決まるまでは「判断に迷う操作は個人で行わず、ICT担当や管理職に相談する」**という原則を伝えておくと安全です。

研修の場で「現在のルールではカバーできていない部分」が明らかになった場合は、それを管理職に報告し、ルール整備の検討材料とすることも研修の成果のひとつです。

### 自治体・設置者のガイドラインとの関係

校内ルールの上位には、教育委員会や設置者が定めるガイドラインがあります。研修の内容がこれらの上位方針と整合しているか、事前に管理職と確認しておくことが重要です。

## 研修計画の例

以下は、1学期に3回の校内研修を実施する場合の計画例です。学校の実情に合わせて時期・内容・回数を調整してください。

### 年間計画の位置づけ（1学期・3回構成）

| 回 | 時期 | テーマ | 対象 | 形式 | 時間 |
|---|---|---|---|---|---|
| 第1回 | 5月中旬 | ICT活用の基本と校内ルール確認 | 全教員 | 講義＋体験 | 60分 |
| 第2回 | 6月下旬 | 生成AIで校務文書の下書きを作る | 希望者＋各学年代表 | ハンズオン | 60分 |
| 第3回 | 7月中旬 | 1学期の実践共有とフォローアップ | 全教員 | 報告＋交流 | 45分 |

### 第1回の詳細例：ICT活用の基本と校内ルール確認（60分）

**ゴール：** 校内のICTルールを全員が理解し、端末の基本操作を1つ確認する

| 時間 | 内容 | 備考 |
|---|---|---|
| 0〜5分 | 校長あいさつ・研修の目的説明 | 管理職が「学校として取り組む」姿勢を示す |
| 5〜15分 | 校内ICTルールの確認 | 端末利用・個人情報・パスワード管理のルール |
| 15〜20分 | 今年度のICT活用方針の共有 | ICT担当から簡潔に説明 |
| 20〜45分 | 体験：端末の基本操作を試す | クラウドへのファイル保存、画面共有の操作など |
| 45〜55分 | グループ共有：感想と質問 | 学年ごとに3〜4人で短く交流 |
| 55〜60分 | まとめ・ミニ課題の提示 | 「今週中に1回、授業で画面提示を試す」 |

### 第2回の詳細例：生成AIで校務文書の下書きを作る（60分）

**ゴール：** 生成AIの基本操作を体験し、安全な使い方のルールを理解する

| 時間 | 内容 | 備考 |
|---|---|---|
| 0〜5分 | 導入：生成AIとは何か（簡潔に） | 検索との違いを1分で説明 |
| 5〜15分 | 安全面の確認 | 入力してはいけない情報・アカウント管理・出力の確認責任 |
| 15〜45分 | 体験：架空の学級通信の下書きを作る | **個人情報は一切使わず、架空のテーマで実施** |
| 45〜55分 | 共有：作成した下書きを見せ合い、気づきを交流 | 「使えそうな点」と「注意が必要な点」の両面を出す |
| 55〜60分 | まとめ・校内ルールの再確認・ミニ課題 | 「次の2週間で、架空題材で1回AIを使ってみる」 |

**体験で使うプロンプトの例（研修資料に記載する）：**

```
あなたは小学校の教員です。
6月の学年だより（保護者向け）の冒頭あいさつ文を200字で作成してください。
テーマ：梅雨の季節の過ごし方
文体：丁寧で温かみのある敬語
※これは研修用の架空の練習です。実在の学校・児童の情報は含みません。
```

### 第3回の詳細例：1学期の実践共有とフォローアップ（45分）

**ゴール：** 研修後の実践経験を共有し、2学期に向けた課題を整理する

| 時間 | 内容 | 備考 |
|---|---|---|
| 0〜5分 | 導入：1学期の振り返り | |
| 5〜25分 | 実践報告：3〜4名が各5分で報告 | うまくいった例も困った例も歓迎 |
| 25〜40分 | グループ交流：自分の実践と課題 | 学年・教科をまたいだグループで交流 |
| 40〜45分 | まとめ：2学期に取り組みたいテーマ | アンケートで希望を収集 |

この3回構成は一例です。学校の規模や年間行事の状況に応じて、回数を減らしたり、短時間のミニ研修に分割したりすることも有効です。大切なのは**「研修→実践→共有」のサイクルを途切れさせないこと**です。

### 2学期以降に発展させる場合

1学期の3回で基礎を固めた後、2学期以降は次のようなテーマに発展させることが考えられます。

- 教科別のICT活用実践共有（国語・算数・理科など教科に特化した事例）
- 生成AIを使った授業準備の効率化（テスト問題の素案作成・ワークシート設計など）
- 児童生徒が端末を使う場面でのルールづくりと指導の工夫
- 保護者への情報発信におけるICT活用（学年だよりのデジタル配信など）
- 校務分掌ごとの効率化アイデア交流会

2学期以降のテーマは、第3回のフォローアップで収集した教員の希望や課題をもとに選ぶと、実態に合った研修になります。

## まとめ

校内でICT・AI研修を進めるためのポイントを整理します。

1. **研修の目的を1つに絞る** — 「何ができるようになるか」を明確にして設計する
2. **体験の時間を確保する** — 聞くだけの研修は定着しない。全体の半分以上を体験に充てる
3. **初心者を置いていかない** — スキル差を前提に、安心して参加できる場をつくる
4. **安全面をセットで扱う** — 特に生成AI研修では、個人情報の入力禁止・出力の確認責任・アカウント管理を必ず扱う
5. **ミニ課題で実践につなげる** — 研修後に「1つだけ試す」仕組みをつくる
6. **フォローアップの場を設ける** — 実践の共有と振り返りで「やりっぱなし」を防ぐ
7. **校内ルール・自治体ガイドラインとの整合を確認する** — 研修内容が既存のルールと矛盾しないよう、事前に管理職と確認する
8. **役割を分担する** — ICT担当だけに任せず、管理職・学年主任・サポーターと連携して進める

ICT活用も生成AIの活用も、最初から完璧にできる必要はありません。研修を通じて「まず1つ試してみる」教員が増えることが、学校全体の変化の第一歩になります。

> 本記事の内容は一般的な研修設計の考え方を整理したものであり、個別の法的判断・医療的判断・行政判断を代行するものではありません。研修の具体的な内容・使用するサービス・運用ルールについては、所属校・教育委員会のガイドラインに従い、管理職と相談の上で決定してください。
