# Buffer CSVアップロード トラブルシューティング

**作成日**: 2026-06-20
**対象**: 千冬先生＠教育DX 初回10投稿 Buffer一括インポート

---

## 発生した現象

- Buffer の「Review Content」画面で **0 valid posts / 10 posts with errors**
- 10件すべてエラー判定

---

## 原因候補

### 原因 1：本文内の改行（最有力）

元CSVは RFC 4180 形式に従い、Textフィールド内に改行を含むダブルクォート囲みを使用していた。
Buffer のCSVパーサーはこの形式に対応していない可能性が高い。

```
# 元CSV（改行あり）
"はじめまして。千冬先生＠教育DXです。
（←ここに改行）
ICT活用・..."
```

→ **簡易版では本文内の改行をすべて削除し、1投稿1行にした。**

---

### 原因 2：Tags列のカンマ区切り値

元CSVのTags列に `site-traffic,ai` のようにカンマ区切りの値を入れていた。
CSVパーサーがフィールド区切りのカンマと混同し、列数不一致エラーになった可能性がある。

```
# 元CSV（Tags列にカンマあり）
"投稿文",,site-traffic,ai,2026-06-23 18:00
# ↑ 5列に解釈される → ヘッダー4列と不一致
```

→ **簡易版ではTags列をすべて空欄にした。**

---

### 原因 3：過去日時

元CSVの1件目が `2026-06-20 07:00`（作業当日の朝）だった。
Bufferは過去日時のスケジュールを拒否する。

→ **簡易版では全件を明日以降の未来日時に設定した。**

---

## 簡易版CSVで変更した内容

| 項目 | 元CSV | 簡易版CSV |
|---|---|---|
| 本文内改行 | あり（段落ごとに改行） | なし（1行に結合） |
| Tags列 | `site-traffic,ai` など | すべて空欄 |
| Image URL列 | 空欄 | 空欄（変更なし） |
| 過去日時 | No.1 が当日朝 | 全件 明日以降に修正 |
| ヘッダー | `Text,Image URL,Tags,Posting Time` | 同じ（変更なし） |
| 文字コード | UTF-8 | UTF-8（変更なし） |

---

## インポート後に確認する項目

1. **Review Content 画面で各投稿のプレビューを目視確認**
   - 10件すべてプレビューが表示されるか
   - 本文が正しく表示されているか（文字化けなし）

2. **URLの確認**
   - サイト誘導投稿（No.2, 4, 6, 8, 10）のURLが正しく表示されているか
   - リンクプレビューが正しく取得されているか

3. **スケジュール日時の確認**
   - Buffer のアカウント設定のタイムゾーンが **Asia/Tokyo（JST）** になっているか
   - 表示される予約日時が日本時間で意図した日時と一致しているか

4. **投稿先アカウントの確認**
   - X（旧Twitter）アカウントが正しく選択されているか

5. **文字数の確認**
   - 各投稿が 280 字以内か（Buffer のカウンターで確認）
   - URLは X 上では 23 字換算になる

## うまくいかない場合：1件だけテスト投稿する手順

### Step 1：テスト用CSVを作成する

以下の内容で `buffer-test-1post.csv` をテキストエディタで作成する。

```
Text,Image URL,Tags,Posting Time
"テスト投稿です。明日削除します。",,,2026-06-22 07:00
```

### Step 2：Buffer にインポートする

1. Buffer → Publish → Queue（またはPlanning）
2. 「Bulk Schedule」または「Import from CSV」を選択
3. 上記テストファイルをアップロード

### Step 3：結果を確認する

- **成功した場合**: 簡易版CSV（10件）を同じ手順でインポートする
- **失敗した場合**: エラーメッセージを確認し、Buffer のサポートページを参照する

### Step 4：テスト投稿を削除する

テストが成功した場合は、Queue から予約投稿を削除する。

## Buffer でよくある問題

| 問題 | 対処 |
|---|---|
| タイムゾーンのズレ | Buffer の Settings → Timezone を Asia/Tokyo（UTC+9）に設定 |
| 文字数オーバー | Buffer の投稿エディタでカウンターを確認（URLは23字換算） |
| 特殊文字によるエラー | 絵文字・制御文字を含む行を取り除いてテスト |
| プランの投稿数上限 | Buffer の無料プランは最大10件程度の制限がある場合がある |
| CSVの文字コード問題 | Excel で CSV を保存すると UTF-8 以外になる場合がある。テキストエディタで保存し直す |

---

## 注意事項

- Buffer の仕様はアップデートで変更されることがある。最新の公式ドキュメントを確認すること。
- インポートした投稿は、必ず送信前に目視確認すること。
- 自動送信が設定されている場合、インポート後すぐに送信される可能性があるため注意。
