# EduAI Platform 開発計画

## 1. 目的と方向性

EduAI Platformは、
- 特別支援DXラボ
- 教育情報DB

を軸に、教育現場・保護者・支援者が必要とする情報を、AI支援と信頼性あるデータでつなぐ教育メディア基盤です。

半年以内に、以下を目指します。
1. 初期ユーザー獲得（教育現場・保護者・支援者）
2. コンテンツ品質と検索体験の向上
3. 広告・アフィリエイト・SaaS提供などの収益化の初期実装

---

## 2. 市場性（PM / SEO Growth / Monetization視点）

### 2.1 市場の背景
- 特別支援教育・発達支援に関する情報は、自治体・学校・支援団体・保護者の間で断片的。
- 「何を調べればよいか分からない」「信頼できる情報源が分散している」という課題が大きい。
- AI検索・要約・FAQ化により、情報アクセスの利便性を高められる。

### 2.2 ターゲット
- 保護者・家庭支援者
- 教育現場の教員・支援員
- 学校・自治体・NPO・福祉・医療関連機関
- 教育メディア運営者・コンテンツ制作会社

### 2.3 市場性の評価
- ニーズの強さ：高
- 情報の継続更新性：高
- 競争優位性の出しやすさ：中〜高
- 収益化の可能性：高（広告・提携・会員・コンサル・SaaS）

### 2.4 競争優位の軸
- 特別支援・教育分野に特化した信頼性高い情報DB
- AIによる要約・比較・検索・FAQ生成
- 学校・自治体・支援機関に向けた実務的なコンテンツ
- SEOに強い構造化データと更新頻度

### 2.5 収益化モデル（初期〜中期）
- ① 広告（Google AdSense / 代理店 / スポンサー）
- ② アフィリエイト（教材・支援ツール・研修・行政資料）
- ③ 有料会員・プレミアムDB
- ④ 教育機関向けAPI / ダッシュボード / 内部検索SaaS
- ⑤ 企業・自治体向けコンサル・導入支援

---

## 3. 技術構成（Fullstack Engineer視点）

### 3.1 基本方針
- 高速な情報発信と検索体験を優先
- SEO・AI検索・コンテンツ更新を容易にする構成
- 将来的な収益化と拡張に耐えるモジュラー設計

### 3.2 推奨構成

#### フロントエンド
- Next.js (App Router)
- TypeScript
- Tailwind CSS
- shadcn/ui
- SEO最適化（metadata, sitemap, structured data）

#### バックエンド
- NestJS または FastAPI
- TypeScript / Python の併用は初期段階では TypeScript優先
- REST API + GraphQLは不要、シンプルなRESTで開始

#### データベース
- PostgreSQL（主データ）
- Redis（キャッシュ / セッション）
- Vector DB（必要時：pgvector / Qdrant）

#### コンテンツ・検索
- CMS: Sanity / Contentful / Notion + API連携
- 検索: Meilisearch / Algolia / PostgreSQL full text
- AI: OpenAI API / Azure OpenAI

#### インフラ
- Vercel（Next.js）
- Railway / Render / Azure App Service（API）
- Supabase / Neon（DB）
- GitHub Actions（CI/CD）

#### セキュリティ・運用
- Auth: NextAuth.js / Clerk
- RBAC（管理者・編集者・閲覧者）
- Sentry / Logflare / CloudWatch
- 監査ログ・バックアップ

### 3.3 主要機能
- 教育情報DB検索
- 特別支援関連カテゴリ別ナビゲーション
- AI要約・FAQ生成・関連資料提案
- 管理画面によるコンテンツ更新
- SEO最適化済み記事一覧・詳細ページ
- ダッシュボード（PV / CTR / 収益）

## 5. リスクと対策

### 5.1 情報信頼性リスク
リスク：支援分野は誤情報の影響が大きい
対策：
- 出典付きコンテンツ運用
- 管理者承認フロー
- 専門家レビュー導入

### 5.2 SEO初期の流入不足
リスク：公開直後に検索流入が伸びにくい
対策：
- キーワード設計を先行
- コンテンツ更新頻度を高める
- 内部リンクと構造化データを最適化

### 5.3 AIの誤回答リスク
リスク：AIの要約が不正確または誤解を招く
対策：
- AI回答に出典リンクを必ず付与
- 「AI生成の補助」として位置づける
- 人手レビューを必須化

### 5.4 収益化の遅れ
リスク：広告や会員化が想定より遅れる
対策：
- 初期からアフィリエイト導線を組み込む
- 早期に提携先候補を複数確保
- 収益化の小さな成功事例を優先する

### 5.5 運用負荷の増大
リスク：記事更新・レビュー・AI運用が重い
対策：
- テンプレート化
- 管理画面の簡素化
- 更新ルールと運用マニュアル整備

## 6. 推奨実行方針

### 優先順位
1. まずは信頼性の高いコンテンツ基盤を作る
2. 次に検索体験とAI補助を強化する
3. 最後に収益化を段階的に拡張する

### 成功指標（半年目）
- 月間PV 1,000〜10,000の段階的達成
- 主要キーワードで検索表示の安定化
- 初期収益化（広告・提携）を実測
- 導入候補・協業候補の確保

## 7. 結論
EduAI Platformは、単なる教育メディアではなく、
「特別支援・教育情報の信頼基盤」と「AI支援検索体験」を持つプラットフォームです。

半年以内に、
- 品質の高い情報提供
- 検索流入の安定化
- 収益化の初期実装

を達成することで、教育メディアとしての成長と事業化の土台を作れます。
