# OPS_NOTES.md — 運用の補足（教育DXナビ・レビュー指摘対応）

最終更新: 2026-07-02 / 担当: Operation Team
既存ファイルは変更せず、運用上の確定事項をここに集約（レビューの軽微指摘4点への回答）。

## 1. status は「2段運用」に統一
- `POSTING_QUEUE.csv` の status は **draft → posted の2段**で運用する（`ready` は使わない）。
- 投稿準備ができたら中間状態を作らず、実際に投稿した時点で `posted` に更新。CSVは全 draft スタートでOK。

## 2. 投稿頻度は「週4本」を基本に（無理なら間引き）
- 生成スケジュールは月〜土の週6本だが、1人運用の継続性を優先し**基本は週4本（例：月・火・木・金）**。
- 余裕がある週だけ土曜の「まとめ/告知」を追加。継続 > 本数。

## 3. 無料PDFの受け取り導線（配布リンクの置き場所）
- 配布ファイル：`学校で生成AIを使う前に確認したいチェックリスト_教育DXナビ.pdf`。
- 置き場所（1つに確定）：**サイト内に配布ページを1枚用意**（例 `/download/ai-checklist`）し、そこにPDFへの直リンクを置く。X/noteからは「記事→配布ページ」で誘導（PV回遊とAdSense面通過を優先。直リンク配布はしない）。
- 未実装の間は、PDF配布を前面に出す投稿は保留し、記事送客の投稿を回す。

## 4. 投稿ID・ログの対応
- `POSTED_LOG.csv` の `post_ref` は **`POSTING_QUEUE.csv` の id（Q01…）を使用**。在庫プール（x_posts_by_category.csv の X-AI-01 等）から採用したものは、採用時に POSTING_QUEUE へ id を採番してから記録する。

## 5. KPIの記入
- `KPI/kpi_content.csv` に例行を追加済み（※実データに置換）。週1でPV・記事別流入・X反応を記入。
