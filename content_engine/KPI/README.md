# KPI記録の使い方

最終更新: 2026-07-02 / 担当: コンテンツ班

`kpi_content.csv` に、投稿ごと／日ごとの数値を記録します。**実測値のみ**を入れてください。分からない数値は空欄のままで構いません（推測やダミー値は入れない）。

## 列の定義

| 列 | 意味 | 入れ方 |
|----|------|--------|
| `date` | 記録日 | `2026-07-06` の形（YYYY-MM-DD） |
| `platform` | プラットフォーム | `X` または `note` |
| `posts` | その日の投稿本数 | 数値 |
| `impressions` | 表示回数（インプレッション） | Xのアナリティクス等の実数。無ければ空欄 |
| `reactions` | いいね・リポスト・スキ等の反応数の合計 | 数値 |
| `clicks` | リンククリック数 | 分かる範囲で。無ければ空欄 |
| `site_or_kindle` | 送客先。ここでは基本 `site`（サイトへの誘導） | `site` |
| `pdf_dl` | 無料PDFのダウンロード数 | 分かる範囲で。無ければ空欄 |
| `notes` | 気づき・特記事項 | 自由記述 |

## 記録のコツ
- 完璧に埋めようとしないこと。続けることが優先です。
- 週末に、その週の行を見て `WEEKLY_REPORTS` を1枚書きます。
- 反応のよかったテーマ・時間帯は `notes` に残しておくと、次の投稿計画に活きます。

## 記入例（イメージ／※これは例。実データに置き換える）
```
2026-07-06,X,1,,,,site,,初回。所見テーマ
```
（impressions等が取れたら後から追記してOK）
